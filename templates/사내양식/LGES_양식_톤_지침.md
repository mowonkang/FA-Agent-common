# LGES 양식 톤 지침

`templates/사내양식/` 의 실제 LGES 표준 양식 3개를 분석하여 도출한 디자인 톤·규칙 모음. 새 양식을 만들거나 본 양식들을 채울 때 본 지침을 따르면 LGES 임원 보고 톤이 유지됩니다.

> ⚠️ 본 지침의 톤·색·사이즈·레이아웃은 **참고 가이드**이며, 가독성·스토리
> 향상을 위해 자율 조정할 수 있습니다. 단 산출물 텍스트가 박스를 벗어나선
> 안 되며, 임원 보고 즉시 활용 품질이 최우선입니다. 임원 보고 12종 양식의
> 블록·placeholder 단서는 [`보고양식_카탈로그.md`](./보고양식_카탈로그.md)
> 참조.

분석 대상:
1. **`260511_인프라그룹 AI경진대회 과제_양식.pptx`** — 한 장 요약형 (개선 전·후 4사분면)
2. **`생산기술혁신센터 MRM 과제 요약서 양식_AX 과제 포함.pptx`** — 과제 요약서 (정보 밀도 매우 높음)
3. **`회의록.pptx`** — 회의록 (대형 본문 폰트)
4. **`(보고양식)YYMMDD_보고제목_v1.0.pptx`** — 임원 보고 12종 양식 모음 (보고양식_카탈로그 참조)

## 1. 공통 톤 (반드시 따를 것)

### 1-1. 슬라이드 사이즈

**27.52cm × 19.05cm** (가로 × 세로, EMU 환산 기준).
- 표준 16:9 와이드 (33.87×19.05) 가 **아님** — 좌우가 좁고 정보 밀도가 높은 LGES 그룹 표준.
- 화면비 약 1.445:1 — 4:3 (1.33) 와 16:9 (1.78) 의 중간.
- 새 양식·신규 첨부 슬라이드는 반드시 이 사이즈로 생성.

```python
from pptx.util import Cm
prs.slide_width  = Cm(27.52)
prs.slide_height = Cm(19.05)
```

### 1-2. 폰트 패밀리

| 용도 | 폰트 | 비고 |
|---|---|---|
| **한글 본문** | `LG스마트체 Regular` | 회사 지정 |
| **영문·숫자·강조** | `Arial Narrow` | 좁은 자폭, 표·라벨에 적합 |
| 한글·영문 혼용 시 | 한글은 `LG스마트체`, 영문·숫자는 `Arial Narrow` 로 자동 분리 | run-level 로 폰트 다르게 지정 |

세 양식 모두 이 두 폰트만 사용. 다른 폰트 (Malgun Gothic, 나눔, 본명조 등) 절대 금지.

### 1-3. 색상 팔레트

| 색상 | HEX | 용도 |
|---|---|---|
| **검정** | `#000000` | 본문 텍스트의 절대 다수 (>80%) |
| **다크 그린** | `#006600` | 강조 — 과제명·부제·책임자·주석 (절제해서 사용) |
| **노랑** | `#FFFF00` | placeholder 자리표시 (사용자가 채워야 할 셀 강조) |
| **그레이** | `#C0C0C0` | 표 헤더·구분선 |
| 흰색 | `#FFFFFF` | 도형 채우기 (본문 영역) |

핵심 원칙:
- **본문은 무조건 검정**. 그린은 제목·라벨에만 절제해서.
- 노랑은 양식의 "여기 채우세요" 표시. 채워진 후에는 검정으로 바꿔야 함.
- **무지개색·블루·레드 등 다른 강조색 일절 금지**. LGES 톤이 깨짐.

### 1-4. 폰트 크기 표

| 크기 | 용도 |
|---|---|
| **20pt** | 큰 표제 (양식 최상단 제목) |
| **16pt** | 슬라이드 메인 제목 (예: "과제 명 : ㅇㅇㅇ") |
| **14pt** | 카드/라벨 헤더 (예: "이슈 및 배경", "기대 효과") |
| **13pt** | 회의록 본문 (정보량 많은 본문에 사용) |
| **12pt** | 일반 본문 bullet, 우상단 책임자 표시 |
| **11pt** | 보조 설명 |
| **10pt** | 표 셀, 부가 설명 |
| **9pt** | 각주 |
| **8pt** | 매우 작은 주석 |
| **7pt** | 최소 — 단위·약어 등 (가독성 한계) |

가장 흔한 크기: **10pt (표·작은 본문)**, **12~13pt (본문)**, **14pt (헤더)**.

### 1-5. 굵기·기울임

- **굵게(bold)**: 헤더·라벨·강조 텍스트
- **이탤릭**: 사용 안 함
- **밑줄**: 사용 안 함 (LGES 양식에 거의 없음)

## 2. 레이아웃 패턴

### 2-1. 한 장 요약형 (개선 전·후 4사분면)

`260511_인프라그룹 AI경진대회 과제_양식.pptx` 형식.

```
┌────── 과제명 : ㅇㅇㅇ (16pt B 검정+그린) ──────┐
│ 부제 (14pt 그린 Arial Narrow)               │  우상단: 인프라.기획팀 OOO 책임 (12pt B 그린)
└─────────────────────────────────────────────┘
┌─── 개선 전 ─────┐ ┌─── 개선 후 ─────┐
│                  │ │                  │
│ [이슈 및 배경]   │ │ [개선 사항]      │
│  • OOO (12pt)    │ │  • OOO           │
│  • OOO           │ │  • OOO           │
│                  │ │                  │
│ [기대 효과]      │ │ [참여 인원] (표) │

│  • OOO           │ │  1| | |          │
```

규칙:
- 좌(현재) ↔ 우(개선) 비교 구조
- 각 사분면은 **라벨 박스 + 본문 bullet 박스** 쌍
- bullet 은 ` • ` (앞 공백 1칸 + 가운데점) 으로 시작

### 2-2. 정보 밀도 높은 과제 요약서

`생산기술혁신센터 MRM 과제 요약서 양식_AX 과제 포함.pptx` 형식.

특징:
- GROUP 도형 다수 (10개) — 도형이 미리 그룹핑되어 있음
- 폰트 크기 7~20pt 광범위 사용
- 표 3개 — 일정·예산·KPI 등
- 노랑 placeholder 3개 — 사용자가 채울 자리 강조
- 그린 사용 **1회만** (매우 절제됨)

규칙:
- 정보가 많을수록 **그린 사용은 줄임**. 검정 단일 톤이 LGES 표준.
- 표가 많을 때는 표마다 헤더(10pt B)·본문(9~10pt) 통일

### 2-3. 본문 중심 (회의록)

`회의록.pptx` 형식.

특징:
- 1 슬라이드 — 회의 1건이 한 장 안에 모두 들어감
- Arial Narrow 111회 — 영문 폰트 압도적
- 13pt 가 본문 표준 (회의록은 다른 양식보다 큼)
- 그린은 부제·강조 5회만

규칙:
- 회의록은 **본문 13pt** (다른 양식의 12pt 보다 큼). 가독성 우선.
- 안건·논의·결정사항·액션아이템 네 영역으로 구분

## 3. placeholder (자리표시) 패턴

LGES 양식은 `{{ key }}` 명시형 placeholder 를 거의 쓰지 않습니다. 대신:

### 3-1. 텍스트 패턴 (가장 흔함)
| 패턴 | 의미 |
|---|---|
| `ㅇㅇㅇ` | 한글 자리표시 (3 글자) |
| `OOO` | 영문/숫자 자리표시 (대문자 O 3개) |
| `xx:00` | 시간 자리표시 |
| `20YY년`, `YY.MMDD`, `MM월DD일` | 날짜 자리표시 |
| `(요일)` | 요일 자리표시 |
| `요약 내용 N`, `안건 N` | 번호 매겨진 항목 |
| 단독 `이름`, `장소`, `회의실` | 단일 단어 자리표시 |

### 3-2. 색상 강조 (보조 신호)
| 색상 | 의미 |
|---|---|
| `#FFFF00` (노랑) | 사용자가 채워야 할 셀 |
| `#FFC000` (주황), `#FF0000` (빨강) | 강조 자리표시 (덜 흔함) |
| 그린 `#006600` 텍스트 | 양식 안내 텍스트 (`* 개인 참여시 1명 만 기입` 같은) |

### 3-3. 채움 후 규칙
- 채운 후에는 **반드시 검정 (#000000)** 으로 색을 바꿈
- 노랑 강조 fill 은 채운 후 제거 (또는 흰색으로)
- 안내 텍스트(`* ...`) 는 양식의 일부로 유지

## 4. 표(table) 규칙

세 양식 모두 표를 핵심 도구로 사용.

### 4-1. 헤더 행
- **10pt Bold**
- 가운데 정렬
- 셀 배경: `#C0C0C0` (그레이) 또는 흰색
- 텍스트: 검정

### 4-2. 데이터 행
- **10pt Regular**
- 숫자·번호는 가운데, 텍스트는 좌측 정렬
- 빈 셀은 양식 그대로 유지 (`　` 전각 공백이 있을 수 있음)
- 행 높이 약 0.65cm

### 4-3. 표 위치
- 슬라이드 우측 또는 하단 (메인 영역 우측 절반·하단 절반)
- 표 너비는 약 12~13cm (양식 폭의 절반)

### 5-1. 우상단

### 5-3. 각주
- `* <내용>`
- **9pt + 그린**
- 슬라이드 하단

|---|---|
| `Rectangle` | 라벨 박스, 카드 외곽 |

- 선 굵기 **0.5pt~0.75pt** (얇게)
- 선 색 검정 또는 그린
- 그림자·블러·그라데이션 사용 안 함 (LGES 평면 디자인)

### 6-3. 채우기
- 본문 영역은 흰색 (`#FFFFFF`) 또는 theme bg
- 라벨 박스는 흰색 또는 검정(라벨 안이 흰 글씨)
- 강조 셀만 그레이/노랑

## 7. python-pptx 구현 팁

### 7-1. 한글 NFD 파일명 처리

LGES 양식 파일명에 한글이 들어가면 macOS/Linux 디스크 인코딩 (NFD) 와 Python 문자열 (NFC) 가 달라 매치 실패.

해결:
```python
import glob
# 한글 부분 없이 ASCII 접두/접미만으로 매치
path = glob.glob("templates/사내양식/260511*.pptx")[0]
```

### 7-2. 도형 텍스트 교체 (폰트 보존)

`shape.text_frame.text = "..."` 는 모든 paragraph 가 합쳐지고 폰트 형식이 리셋됨. 폰트 보존하면서 텍스트만 갈아끼우려면:

```python
def replace_textframe(tf, lines, *, font="LG스마트체 Regular",
                      size=12, bold=False, color=BLACK):
    tf.clear()
    for i, line in enumerate(lines):
        para = tf.paragraphs[0] if i == 0 else tf.add_paragraph()
        run = para.add_run()
        run.text = line
        run.font.name = font
        run.font.size = Pt(size)
        run.font.bold = bold
        run.font.color.rgb = color
```

### 7-3. 표 셀 텍스트 교체

```python
def replace_cell_text(cell, text, *, size=10, bold=False):
    replace_textframe(cell.text_frame, [text],
                      size=size, bold=bold,
                      align=PP_ALIGN.CENTER)
```

빈 셀 (`r2~r5`) 은 그대로 두면 양식의 `　` 전각 공백이 유지됨.

### 7-4. 색상 상수

본 리포의 LGES 톤 상수 (재사용 권장):
```python
BLACK  = RGBColor(0x00, 0x00, 0x00)
GREEN  = RGBColor(0x00, 0x66, 0x00)  # LGES 강조색
WHITE  = RGBColor(0xFF, 0xFF, 0xFF)
GRAY   = RGBColor(0xC0, 0xC0, 0xC0)  # 표 헤더용
YELLOW = RGBColor(0xFF, 0xFF, 0x00)  # placeholder 자리표시
```

`scripts/build_contest.py` 의 색·헬퍼 함수가 본 지침을 그대로 구현하니 참조.

## 8. 첨부·보조 슬라이드 (회색조 톤)

본 양식 톤(검정+그린)을 메인에 유지하고, 첨부·보조 슬라이드는 **단조 회색조** 로 구분하면 가독성이 좋아짐.

회색조 팔레트:
```python
CHARCOAL  = RGBColor(0x2A, 0x2A, 0x2A)  # 본문 (검정보다 약간 부드러움)
DIM_GRAY  = RGBColor(0x4D, 0x4D, 0x4D)  # 라벨·헤더 띠
MID_GRAY  = RGBColor(0x8C, 0x8C, 0x8C)  # 보조 라벨
LIGHT_GRAY = RGBColor(0xF5, 0xF5, 0xF5)  # 카드 배경
SOFT_GRAY  = RGBColor(0xE5, 0xE5, 0xE5)  # 본문 박스
LINE_GRAY  = RGBColor(0xCC, 0xCC, 0xCC)  # 구분선
```

규칙:
- 그린 사용 금지 (첨부에서)
- 노랑·빨강 등 강조색 일절 사용 안 함
- 폰트·크기는 메인과 동일 (LG스마트체 / Arial Narrow)

`scripts/build_contest.py` 의 `add_gray_card`·`add_band`·`add_appendix_titlebar` 헬퍼가 이 규칙을 그대로 구현.

## 9. 체크리스트

새 양식·신규 슬라이드 작성 시:

- [ ] 슬라이드 사이즈 **27.52 × 19.05 cm**
- [ ] 폰트는 **LG스마트체 Regular** (한글) + **Arial Narrow** (영문/숫자) 만 사용

- [ ] 사용자가 채울 자리는 노랑 (#FFFF00) 또는 `ㅇㅇㅇ`/`OOO` 텍스트로 표시
- [ ] 본문 폰트 12pt 또는 13pt, 헤더 14pt, 표 셀 10pt, 각주 9pt
- [ ] bullet 은 ` • ` (앞 공백 1칸 + 가운데점)
- [ ] 우상단에 "조직.팀 OOO 책임" 식 책임자 표기 (12pt B 그린 Arial Narrow)
- [ ] 표 헤더는 10pt Bold 가운데 정렬, 데이터는 10pt Regular
- [ ] 그림자·그라데이션·블러·블루/레드/오렌지 강조색 사용 안 함
- [ ] 첨부는 회색조 (검정+그린 사용 안 함) — 본 양식과 시각적으로 명확히 구분

## 10. 분석 데이터 (참고)

### 텍스트 색상 빈도 (3개 양식 통합)

| HEX | 빈도 (합) |
|---|---|
| `#000000` 검정 | 51회 |
| `#006600` 그린 | 26회 |

### 폰트 빈도

| 폰트 | 빈도 (합) |
|---|---|
| `Arial Narrow` | 188회 |
| `LG스마트체 Regular` | 32회 |

영문 폰트가 압도적인 이유: 라벨·표 헤더·번호·책임자 표기가 대부분 영문/숫자/특수문자 비중이 높아 Arial Narrow 로 처리됨.

### 슬라이드당 도형 수
