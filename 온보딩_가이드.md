# FA-Agent 온보딩 가이드 — 처음 쓰는 구성원용

> **이 문서 하나로 시작할 수 있습니다.** GitHub? Claude? 코딩? **전혀 몰라도 됩니다.**
> 계정을 만드는 첫 단계부터, 한국어로 일을 시키고 완성된 보고서를 받는 단계까지 순서대로 안내합니다.

| 항목 | 내용 |
|---|---|
| 대상 | FA기술혁신 Part 전 구성원 (비개발자 포함) |
| 준비 시간 | 처음 1회 약 30분 (계정 2개 만들기) |
| 첫 사용 | 5분 (한국어 한 문장이면 됩니다) |
| 관리자/리더 | 신규 구성원 등록 담당자는 **10장 관리자 부록**을 먼저 보세요 |

**3줄 요약**

1. FA-Agent는 **"한국어로 말하면 AI 팀원 5명이 우리 팀 양식대로 보고서를 만들어 주는 시스템"** 입니다.
2. 처음 1회만 **계정 2개**(GitHub, Claude)를 준비하면, 그 다음부터는 **채팅하듯 일을 시키면** 됩니다.
3. 명령어·코드를 외울 필요가 없습니다. *"이 녹취록으로 회의록 PPT 만들어줘"* 처럼 평소 말투로 쓰면 됩니다.

## 1. FA-Agent가 뭔가요? — 3분 개념 잡기

### 1.1 등장인물은 딱 3개입니다

| 이름 | 비유하자면 | 하는 일 |
|---|---|---|
| **Claude(클로드)** | 말귀를 알아듣는 AI 직원 | 한국어를 이해하고 글·문서·분석을 해 주는 인공지능 |
| **Claude Code(클로드 코드)** | 그 AI 직원의 **사무실** | Claude가 우리 팀 폴더(양식·데이터·자료)를 직접 열어 보고, 파일을 만들어 주는 작업 환경 |
| **GitHub(깃허브)** | 우리 팀의 **공유 캐비닛** | 양식·데이터·결과물이 보관되는 인터넷 저장소. 누가 언제 무엇을 바꿨는지 자동 기록됨 |

> 즉, **GitHub 캐비닛 안에 우리 팀 자료가 있고, Claude Code 사무실에서 Claude 직원이 그 자료로 일을 합니다.**
> 여러분이 할 일은 그 직원에게 **한국어로 업무 지시**를 하는 것뿐입니다.

### 1.2 우리 팀의 AI 팀원 5명

FA-Agent에는 역할이 나뉜 AI 팀원 5명이 세팅되어 있습니다. **누구를 부를지 여러분이 고를 필요는 없습니다** — 팀장(메인 Claude)이 알아서 적임자에게 일을 나눕니다. (출처: CLAUDE.md "에이전트 5명" 절)

| AI 팀원 | 역할 (비유) | 담당 |
|---|---|---|
| `data-teammate` | 데이터 담당 사원 | KPI 진척 · Capex(투자비) · MRM(양산 일정) · DST(R&D 진척) 숫자 조회 |
| `ops-teammate` | 운영 담당 사원 | 협력사 현황 · HR(출장/교육/포상) · 결재 대기 · 사내문서 검색 |
| `tech-research-teammate` | 리서치 담당 사원 | 외부 기술 동향 · 경쟁사 정보 · 신규 과제 발굴 (AMR/협동로봇/디지털트윈/스마트물류/AI/피지컬AI) |
| `document-writer` | 문서 작성 담당 | 회의록·주간/월간보고 등 **markdown 문서** 양식 채움 |
| `ppt-writer` | PPT 담당 | 사내 **PPT 양식**을 분석해 빈칸을 채운 PPTX 생성 (LGES 표준 색·폰트 자동 적용) |

### 1.3 폴더는 4개만 기억하세요

| 폴더 | 의미 |
|---|---|
| `templates/` | **양식 보관함** — 회의록·보고서의 빈 양식 (원본은 절대 수정되지 않음) |
| `sample_data/` | **숫자 데이터** — KPI, 협력사, 투자비 등 |
| `references/` | **참고 자료** — 로드맵·정책·과거 회의록·기술자료·용어집 |
| `outputs/` | **결과물 수령함** — AI가 만든 완성본은 전부 여기 생깁니다 |

### 1.4 무엇을 시킬 수 있나요? (대표 업무)

- 녹취록을 주면 → **회의록 PPT/Word 자동 작성**
- "주간보고 써줘" → 데이터 팀원이 숫자를 모으고, 작성 팀원이 **양식대로 채움**
- "KPI 진척 알려줘" → **출처(파일명·행 번호)와 함께** 답변
- "FA 기술 동향 조사해줘" → 외부 자료를 조사해 **과제 후보 보고서** 작성
- 엑셀 정리, PDF에서 표 추출, Word 문서 작성, 임원 보고용 PPT 등

## 2. 처음 1회 준비 — 계정 2개 만들기

> 이 장은 **딱 한 번만** 하면 됩니다. 이미 계정이 있다면 해당 단계는 건너뛰세요.
> 막히면 혼자 고민하지 말고 관리자([10장](#10-관리자-부록--신규-구성원-등록과-자료-관리) 담당자)에게 요청하세요.

### 2단계 요약

```
① GitHub 계정 만들기 → 관리자에게 초대 요청 → 초대 수락
② Claude 계정 만들기 → 회사 시트(라이선스) 배정 받기
```

### 2.1 GitHub 계정 만들기 (5분)

1. 인터넷 브라우저에서 **github.com** 접속 → 우측 상단 **Sign up** 클릭
2. 이메일 주소, 비밀번호, 사용자 이름(영문)을 입력해 가입
   - 사용자 이름은 동료가 알아볼 수 있게 (예: `hong-gildong-lges`)
3. 가입 확인 메일의 인증 절차를 완료

### 2.2 팀 저장소 초대 받기 (1분 + 대기)

1. 관리자에게 **GitHub 사용자 이름**을 알려주고 `FA-Agent-common` 저장소 초대를 요청
2. 초대 메일이 오면 **View invitation → Accept invitation** 클릭
3. 브라우저에서 `github.com/mowonkang/FA-Agent-common` 이 열리면 성공
   - 이 페이지가 바로 우리 팀의 "공유 캐비닛"입니다. 눌러보다가 망가뜨릴 걱정은 하지 않아도 됩니다 — 모든 변경은 기록되고 되돌릴 수 있습니다.

### 2.3 Claude 계정 만들기 (5분)

1. 브라우저에서 **claude.ai** 접속 → 회사 이메일로 가입
2. **Claude Code 사용 권한이 포함된 요금제/시트가 필요합니다.**
   - 회사(팀) 계약이 있는 경우: 관리자에게 시트 배정을 요청하세요
   - 개인 결제로 시작하는 경우: Pro 이상 요금제에서 Claude Code 사용 가능
   - *[우리 회사 라이선스 신청 절차: 담당자가 기입해 주세요]*

### 2.4 준비 완료 체크리스트

- [ ] github.com 에 로그인할 수 있다
- [ ] github.com/mowonkang/FA-Agent-common 페이지가 열린다
- [ ] claude.ai 에 로그인할 수 있다
- [ ] (회사 계약 시) Claude Code 권한이 있는 시트를 배정받았다

여기까지 됐다면 준비 끝입니다. 다음 장에서 사용 방법을 고릅니다.

## 3. 사용 방법 고르기 — 웹(추천) vs PC 설치

두 가지 길이 있습니다. **처음이라면 무조건 방법 A(웹)를 추천합니다** — 설치할 것이 하나도 없습니다.

| | 방법 A: 웹에서 사용 ★추천 | 방법 B: 내 PC에 설치 |
|---|---|---|
| 설치 | **없음** (브라우저만) | Claude Code 프로그램 + 팀 폴더 내려받기 |
| 난이도 | 쉬움 | 중간 (안내대로 하면 가능) |
| 결과물 받기 | 화면에서 다운로드 / GitHub에 저장 | 내 PC 폴더(`outputs/`)에서 바로 열기 |
| 어디서나 | PC·노트북·휴대폰(Claude 앱) | 설치한 PC에서만 |
| 추천 대상 | 모든 구성원, 특히 첫 사용자 | 매일 쓰는 파워유저, 관리자 |

### 3.1 방법 A — 웹에서 사용 (설치 0개)

1. 브라우저에서 **claude.ai/code** 접속 (Claude 계정으로 로그인)
2. 처음 1회: 화면 안내에 따라 **GitHub 계정 연결**을 승인
   - "Claude가 GitHub에 접근하도록 허용할까요?"라는 화면이 나오면 **승인(Authorize/Install)** — 우리 저장소를 읽고 쓰기 위한 정상 절차입니다
3. 저장소 목록에서 **`FA-Agent-common`** 선택
4. 채팅창이 열리면 끝 — **여기에 한국어로 일을 시키면 됩니다**

> 💡 휴대폰의 Claude 앱에서도 같은 세션을 이어서 쓸 수 있습니다. 출장 중에 "주간보고 초안 잡아줘"라고 시켜두는 것도 가능합니다.
>
> 💡 웹에서는 AI가 클라우드의 복사본에서 작업하고, 결과를 **GitHub에 저장(PR)** 하거나 **파일로 다운로드**하도록 요청할 수 있습니다. "결과 파일 보내줘" / "GitHub에 저장해줘"라고 말하면 됩니다.

### 3.2 방법 B — 내 PC에 설치 (파워유저/관리자용)

> 비개발자도 따라할 수 있도록 적었지만, 막히면 방법 A로 시작해도 전혀 문제 없습니다.

**B-1. Claude Code 설치**

- **Windows**: 시작 메뉴에서 `PowerShell` 검색 → 실행 → 아래 한 줄을 붙여넣고 Enter

  ```
  irm https://claude.ai/install.ps1 | iex
  ```

- **Mac**: Spotlight(🔍)에서 `터미널` 검색 → 실행 → 아래 한 줄을 붙여넣고 Enter

  ```
  curl -fsSL https://claude.ai/install.sh | bash
  ```

- 설치 후 터미널에 `claude` 를 입력하면 브라우저가 열리며 로그인 안내가 나옵니다 → Claude 계정으로 로그인
- 최신 설치 안내(공식, 한국어 지원): **code.claude.com/docs**

**B-2. 팀 폴더(저장소) 내려받기 — 둘 중 편한 것 하나**

- *클릭만으로*: **GitHub Desktop** 프로그램 설치(desktop.github.com) → `File → Clone repository` → `mowonkang/FA-Agent-common` 선택 → 저장 위치 지정
- *명령 한 줄로*: 터미널/PowerShell에서

  ```
  git clone https://github.com/mowonkang/FA-Agent-common.git
  ```

**B-3. 부가 기능 준비 (선택이지만 권장)**

- **Python 3** 설치 (python.org — Windows는 설치 시 "Add python.exe to PATH" 체크)
- 내려받은 폴더에서 PPT 자동 채움 기능용 패키지 설치: (출처: README.md [149-158행])

  ```
  pip install -r requirements.txt
  ```

**B-4. 실행**

1. 터미널/PowerShell에서 내려받은 폴더로 이동: `cd FA-Agent-common`
2. `claude` 입력 → 대화창이 뜨면 **한국어로 일을 시키면 됩니다**
3. 처음 1회는 AI 팀원 기능(Agent Teams) 적용을 위해 Claude Code를 한 번 껐다 켜 주세요 (출처: README.md [115행])

> 참고: Claude **데스크톱 앱**(Mac/Windows)과 **VS Code 확장**에서도 Claude Code를 쓸 수 있습니다. 이미 쓰는 도구가 있다면 그 안에서 같은 방식으로 동작합니다.

## 4. 첫 사용 5분 튜토리얼

### 4.1 첫 명령 — 이대로 복사해서 붙여넣으세요

팀 세팅이 정상인지 확인하는 공식 테스트 명령입니다. (출처: README.md [106-113행])

```
sample_data/kpi_샘플.md의 Q3 KPI를 회의록 양식으로 정리해줘.
일시는 오늘, 참석자는 테스트 사용자.
```

잠시 후 `outputs/회의록_<오늘날짜>.md` 파일이 생기면 **모든 것이 정상 동작하는 것입니다.**

### 4.2 사용 중 화면에서 만나는 것들

| 상황 | 의미 | 어떻게 하나요 |
|---|---|---|
| **"~를 실행해도 될까요?" 권한 질문** | AI가 파일을 만들거나 프로그램을 돌리기 전에 허락을 구하는 것 | 내용을 읽고 **승인**. 이상하면 거부하고 "왜 필요한지 설명해줘"라고 물어보세요 |
| **계획(Plan)을 보여주며 승인 요청** | 문서를 쓰기 전에 "이렇게 채우겠습니다" 하고 검사받는 것 | 방향이 맞으면 승인, 아니면 "○○는 빼고, △△를 강조해줘"라고 수정 지시 |
| **AI 팀원들이 일하는 표시** | 팀장이 데이터/운영/작성 팀원에게 일을 나눠준 상태 | 기다리면 됩니다 (보통 수 분 이내) |
| **"자료 미확인"이라는 표기** | 근거 자료를 못 찾았다는 **정직한 보고** (지어내지 않음) | 자료 위치를 알려주거나, 그대로 두고 사람이 채우면 됩니다 |

### 4.3 결과물 받기

- 결과물은 **항상 `outputs/` 폴더**에 생깁니다. 파일명은 `양식이름_YYYY-MM-DD.pptx` 형식이고, 같은 날 다시 만들면 `_v2`, `_v3`가 붙습니다 (출처: README.md [186-189행])
- **웹(방법 A)**: "결과 파일 보내줘"라고 하면 다운로드 버튼이 나타납니다. 팀 캐비닛에 보관하려면 "GitHub에 저장해줘(PR 만들어줘)"
- **PC(방법 B)**: 탐색기/Finder에서 `FA-Agent-common/outputs/` 폴더를 열면 바로 있습니다

### 4.4 대화를 이어가는 법

- 결과가 마음에 안 들면 그냥 말하세요: *"두 번째 섹션이 너무 길어. 3줄로 줄여줘"*
- 중간에 멈추고 싶으면 `Esc` 키 (PC) 또는 정지 버튼 (웹)
- 어제 하던 작업을 이어가려면: PC에서는 `claude --resume`, 웹에서는 세션 목록에서 클릭

## 6. 잘 쓰는 요령 — 좋은 요청 vs 아쉬운 요청

AI 팀원은 **신입사원과 같습니다.** 일을 잘 시키는 요령도 신입에게 지시하는 요령과 같습니다.

| 원칙 | 아쉬운 요청 ✗ | 좋은 요청 ✓ |
|---|---|---|
| **목적을 말하기** | `보고서 써줘` | `다음주 월요일 그룹장 보고용 주간보고 써줘` |
| **기준·범위 주기** | `KPI 알려줘` | `Q3 KPI 진척률을 목표 대비로 알려줘` |
| **아는 값은 직접 주기** | (작성자를 비워둠) | `작성자는 김OO, 일시는 6/12 14시로 넣어줘` |
| **형식을 지정하기** | `정리해줘` | `표 1개 + 요약 3줄로 정리해줘` |
| **한 번에 한 묶음** | 회의록+동향조사+엑셀을 한 문장에 | 회의록 먼저 → 끝나면 다음 요청 |

그리고 두 가지를 기억하세요.

- **다시 시키는 것을 어려워하지 마세요.** "다시", "더 짧게", "어조를 공손하게" — 몇 번을 고쳐 시켜도 AI는 지치지 않습니다.
- **모르면 AI에게 물어보세요.** *"네가 뭘 할 수 있는지 알려줘"*, *"이 저장소에 어떤 양식이 있어?"* 라고 물으면 직접 설명해 줍니다. 이 가이드를 잃어버려도 AI에게 물으면 됩니다.

## 7. 꼭 지켜야 할 팀 규칙과 보안

우리 팀 AI는 아래 원칙이 시스템으로 강제되어 있습니다. 사람도 같은 원칙을 따라 주세요. (출처: CLAUDE.md "절대 원칙" 절)

1. **추측 금지** — AI는 자료가 없으면 지어내는 대신 "자료 미확인"이라고 적습니다. 사람이 그 칸을 임의로 채울 때도 근거를 확인하세요.
2. **출처 명시** — AI의 답에는 `(출처: 파일명 [행 번호])`가 붙습니다. **보고 전에 원본 숫자와 대조**하는 것은 사람의 몫입니다. 출처 인용이 없는 보고서는 시스템이 자동으로 차단합니다 (출처: scripts/hooks/check_output_citations.py [1-16행])
3. **원본 보존** — `templates/`(양식)와 `references/`(참고자료) 원본은 수정하지 않습니다. 결과물은 `outputs/`에만 생깁니다.
4. **최종 책임은 사람** — AI 산출물은 "초안"입니다. 결재·보고 전 검토 책임은 작성자에게 있습니다.

**보안 주의사항**

- 현재 `sample_data/`의 데이터는 **시연용 샘플**입니다. 실제 사외비 데이터를 올리기 전에는 반드시 보안/IT 검토를 거치세요.
- 사외비 문서·개인정보를 채팅에 붙여넣기 전에 회사 AI 사용 지침을 확인하세요. *[우리 회사 보안 지침 링크: 담당자가 기입해 주세요]*
- 외부 서비스 연동(웹 스크래핑 키 등)은 보안/IT 검토 후에만 활성화합니다.

## 8. 자주 묻는 질문(FAQ)과 문제 해결

**Q. 영어로 써야 하나요? 명령어를 외워야 하나요?**
아니요. **전부 한국어로, 평소 말투로** 쓰면 됩니다. 외울 명령어는 없습니다.

**Q. 제가 뭘 잘못 눌러서 팀 자료가 지워지면 어쩌죠?**
걱정하지 않아도 됩니다. GitHub는 모든 변경 이력을 보관하므로 **언제든 되돌릴 수 있습니다.** 또한 AI는 파일을 바꾸기 전에 허락을 구하도록 설정되어 있습니다.

**Q. AI가 만든 숫자를 믿어도 되나요?**
답변에 붙는 `(출처: 파일명 [행])`을 따라가 원본을 확인하세요. 출처가 없거나 "자료 미확인"이면 그 값은 보고에 쓰지 마세요.

**Q. "작업 완료가 차단되었습니다(missing_citations)"라고 떠요.**
보고서에 출처 인용이 하나도 없으면 품질 게이트가 막은 것입니다. *"누락된 출처를 추가해줘"* 라고 말하면 해결됩니다. 자료가 정말 없는 항목은 "자료 미확인"으로 표기하게 하세요. (출처: CLAUDE.md "트러블슈팅" 절)

**Q. 어제 세션을 이어갔더니 AI 팀원들이 사라졌어요.**
알려진 제한입니다. *"팀원을 다시 생성해줘"* 라고 말하면 같은 구성으로 다시 만들어집니다. (출처: CLAUDE.md "트러블슈팅" 절)

**Q. PPT를 만들었는데 일부 칸이 비어 있어요.**
근거를 못 찾은 칸은 추측으로 채우지 않고 비워 둡니다(unfilled 목록으로 보고됨). 값을 알려주며 다시 시키거나 직접 채우면 됩니다. 그림 속 글자·SmartArt·차트 내부 수치는 자동 채움 대상이 아닙니다. (출처: README.md [191-196행])

**Q. 참고자료가 오래된 것 같아요 / 새 자료를 넣고 싶어요.**
새 자료는 `references/` 폴더에 넣기만 하면 다음 세션부터 자동 인식됩니다. 급하면 *"참조 인덱스 갱신해줘"* 라고 말하세요. (출처: CLAUDE.md "references/ 사용 규약" 절)

**Q. 응답이 멈췄거나 이상해요.**
`Esc`(또는 정지 버튼)로 멈추고 다시 지시하세요. 그래도 안 되면 새 세션을 열고 *"아까 하던 ○○ 작업 이어서 해줘"* 라고 하면 됩니다.

**Q. 더 배우고 싶어요.**
공식 문서(한국어): **code.claude.com/docs** · 이 저장소의 기술 문서: `README.md`, `CLAUDE.md`

**그 밖의 모든 문제** → 관리자에게 문의: *[담당자 이름/연락처: 기입해 주세요]*

| 용어 | 쉬운 설명 |

| **리포지토리(저장소, repo)** | GitHub의 프로젝트 폴더. 우리 팀 것은 `FA-Agent-common` |
| **클론(clone)** | 저장소를 내 PC로 복사해 내려받는 것 |
| **커밋(commit)** | 변경 내용을 "저장 + 기록"하는 것. 문서의 '버전 저장'과 같음 |
| **푸시(push)** | 내 PC의 변경을 GitHub(공유 캐비닛)에 올리는 것 |
| **PR(Pull Request)** | "이 변경을 팀 자료에 반영해 주세요"라는 **반영 요청서**. 검토 후 합칩니다 |
| **브랜치(branch)** | 원본을 건드리지 않고 작업하는 **임시 사본**. PR로 원본에 합쳐짐 |
| **터미널 / PowerShell** | 글자로 명령을 입력하는 검은 창. 방법 B에서만 잠깐 씁니다 |
| **훅(hook)** | 특정 시점에 자동 실행되는 검사 장치 (예: 출처 누락 시 완료 차단) |
| **플레이스홀더(placeholder)** | 양식의 빈칸 표시 (예: `{{ 작성자 }}`). AI가 이 자리를 채웁니다 |
| **markdown(.md)** | 서식 있는 텍스트 문서 형식. 메모장처럼 열리고 GitHub에서 예쁘게 보임 |
