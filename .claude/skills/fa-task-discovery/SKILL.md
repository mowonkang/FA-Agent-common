# FA 기술과제 발굴 에이전트

## 목적

배터리 공장 FA 담당 엔지니어(모원님)를 위해 6개 기술 분야의 글로벌 동향을 자동 수집하고,
과제화 가능한 아이디어를 도출하여 **docx 보고서**와 **카카오톡 요약**으로 전달한다.

**탐색 분야**: AMR/AGV · 협동로봇 · 디지털 트윈 · 스마트 물류 · AI · 피지컬 AI

**과제 후보 필터 기준**:
1. 배터리 공장(셀/모듈/팩 라인)에 실제 적용 가능한 기술
2. LGES가 아직 공식 도입·양산 적용하지 않은 기술

## 보고서 생성 스크립트

`scripts/generate_report.js` — docx 보고서 생성 스크립트 (Node.js + docx 라이브러리). **1개 파일·2부 구성** docx 를 만든다 (제1부: 최근 1개월 뉴스 브리핑, 제2부: 기술과제 후보).

사용법:
```bash
cd scripts && npm install
node generate_report.js <output.docx> <tasks.json|-> [news.json]
```
- `tasks.json`: `[{num, category, title, trend, apply, reference}, ...]` 형식 (제2부)
- `news.json`: `[{category, title, url, date, summary}, ...]` 형식 (제1부)
  - `category` 는 6개 분야 별칭 허용(예: "AI(제조)"→AI, "휴머노이드"→피지컬 AI), 미매칭은 "기타" 그룹
- **일반 사용 시**: 두 파일 모두 전달 → **제1부 뉴스 + 제2부 과제 2부 docx 1개** 생성
- 폴백 모드 (디버그용):
  - tasks.json 만 → 과제 단일부 docx
  - news.json 만 (tasks 자리에 `-`) → 뉴스 단일부 docx
  - 둘 다 없으면 오류 종료
- 실행 전 `scripts/` 디렉토리에서 `npm install` 필요 (최초 1회)

## 전체 워크플로우

**항상 자동 모드로 동작한다.** 사용자에게 후보를 확인받지 않는다.

**한 번의 실행에서 ① 뉴스 수집 → ② 과제 후보 도출 → ③ 단일 2부 docx 생성 → ④ 단일 PR 자동 생성까지 모두 끝낸다.** 뉴스 브리핑만 만들고 멈추지 않으며, 과제 docx 가 완성된 다음에야 한 번만 commit·push·PR을 수행한다 (사용자 요청).

- 전용 폴더: **`outputs/fa-task-discovery/`** (산출물 모두 여기에 보관)
- 오늘자 파일 규칙(`{YYYYMMDD}` = 실행일):
  - `news_{YYYYMMDD}.json` — 뉴스 원천 (출처 URL·날짜 포함)
  - `tasks_{YYYYMMDD}.json` — 과제 후보 5~10건
  - `FA_기술과제후보_{YYYYMMDD}.docx` — 최종 docx (제1부 뉴스 + 제2부 과제)
- **중간 산출물(news.json, 뉴스만 담은 docx)을 별도 PR 로 올리지 않는다.** 같은 날 재실행 시 news.json 이 이미 있으면 **재수집을 건너뛰고 캐시로 사용**한 뒤 곧장 과제 도출로 진행한다.

```
[실행 트리거 — 수동 호출 / 매일 자동 스케줄]
                  ▼
  ① naver-tech-researcher + tech-web-researcher 병렬 (최근 30일 뉴스)
        ▼
  ② news_{today}.json 저장 (이미 있으면 재사용, 추가 push 없음)
        ▼
  ③ news.json 근거로 과제 후보 5~10건 도출 → tasks_{today}.json
        ▼
  ④ generate_report.js 1회 실행 → FA_기술과제후보_{today}.docx (제1부 뉴스 + 제2부 과제)
        ▼
  ⑤ 날짜 브랜치 1회 commit·push (news.json + tasks.json + 최종 docx 동시)
        ▼
  ⑥ 메인 대상 PR 자동 생성 (중복 PR 방지) — 워드 완성 후 PR은 **1개**
        ▼
  ⑦ KakaotalkChat-MemoChat 요약 전송
```

## 단계별 지침

> **한 번의 실행에서 1단계부터 7단계까지 끝까지 수행한다.** 뉴스만 만들고 멈추지 않으며, docx 가 완성된 뒤에만 1회 commit·push·PR을 호출한다.

### 0단계 — 캐시 확인 (세션 분기 아님)

```bash
REPO_ROOT="<리포 루트>"          # 예: /home/user/claude_FA-Agent_1
DIR="$REPO_ROOT/outputs/fa-task-discovery"
D=$(date +%Y%m%d)
mkdir -p "$DIR"
```
- 오늘자 `news_$D.json` 이 **이미 있으면** 1·2단계(뉴스 수집)를 **건너뛰고** 그 파일을 그대로 재사용해 3단계로 진행 (네트워크 절약, 같은 날 재실행 대응).
- 없으면 1·2단계로 새로 수집한다. **어느 경우든 3·4·5·6·7단계를 반드시 끝까지 수행한다.**

### 1단계 — 두 서브에이전트 병렬 실행 (최근 1개월 뉴스)

**반드시 한 메시지에서 두 Task를 동시에 호출한다.** 오늘 날짜를 `today` 로 두고, **수집 대상은 today 기준 최근 30일(약 1개월) 이내 발행 기사로 한정**한다.

**Task 1** (`naver-tech-researcher`):
```
아래 6개 분야에서 배터리/제조 FA 관련 최신 한국어 뉴스를 수집해줘.
분야: AMR/AGV, 협동로봇, 디지털 트윈, 스마트 물류, AI(제조 적용), 피지컬 AI(휴머노이드·임바디드AI)
★ 오늘(today) 기준 최근 30일(약 1개월) 이내 발행된 기사만. 1개월 초과·발행일 불명 기사는 제외.
★ 각 기사마다 발행일(YYYY-MM-DD)과 출처 URL을 반드시 포함.
각 분야별 최신 기사 3~5건을, 항목별로 {category, title, url, date, summary(핵심 1줄)} JSON 배열로 반환해줘.
추가 검색어: "배터리 공장 자동화", "이차전지 스마트팩토리", "산업용 로봇 최신"
```

**Task 2** (`tech-web-researcher`):
```
아래 6개 분야에서 글로벌 FA/제조 기술 동향을 영어·한국어로 수집해줘.
분야: AMR/AGV, collaborative robot, digital twin, smart logistics, AI in manufacturing, physical AI humanoid
소스: therobotreport.com, automationworld.com, aitimes.com, etnews.com, zdnet.co.kr
★ 오늘(today) 기준 최근 30일(약 1개월) 이내 발행된 기사만. 1개월 초과·발행일 불명 기사는 제외.
★ 각 기사마다 발행일(YYYY-MM-DD)과 출처 URL을 반드시 포함.
각 분야별 최신 동향 3~5건을, 항목별로 {category, title, url, date, summary(핵심 1줄)} JSON 배열로 반환해줘.
```

❌ **순차 호출 금지** — 두 Task를 반드시 같은 메시지에서 동시에 호출.

### 2단계 — news_{today}.json 저장 (출처 포함 최신 뉴스 정리)

두 서브에이전트 결과를 병합하고 **30일 초과·발행일 불명 기사를 버린 뒤**, 6개 분야별로 정리해 **`outputs/fa-task-discovery/news_$(date +%Y%m%d).json`** 으로 저장한다.

```json
{ "category": "AMR/AGV", "title": "기사 제목", "url": "https://...", "date": "2026-05-02", "summary": "핵심 1줄 요약" }
```

이 파일이 docx **제1부(최근 1개월 뉴스 브리핑)** 의 원천이다. 분야별 3~5건 권장. ⚠️ **여기서 멈추지 않는다. 절대 commit·push·PR 을 하지 말고 곧장 3단계로 진행한다.**

### 3단계 — 동향 분석 및 과제 후보 도출

오늘자 **`outputs/fa-task-discovery/news_$(date +%Y%m%d).json`** 을 근거로, 아래 기준으로 과제 후보 5~10개를 도출한다.

- **조건 A**: 배터리 셀/모듈/팩 생산라인, 물류, 검사 공정에 실질 적용 가능한가?
- **조건 B**: LGES 공식 도입·양산 발표가 없는 기술인가?
- **조건 C**: 근거 동향이 최근 1개월(news.json) 내 기사인가? (신선도 보증)

각 후보 구조:
```json
{
  "num": 1,
  "category": "AMR/AGV",
  "title": "과제명",
  "trend": "동향 근거 1~2문장",
  "apply": "배터리 공장 적용 포인트",
  "reference": "글로벌 참고 사례 1건"
}
```

### 4단계 — tasks_{today}.json 저장 (항상 전체 후보)

도출된 전체 과제 후보를 **`outputs/fa-task-discovery/tasks_$(date +%Y%m%d).json`** 으로 저장한다. 사용자에게 선택을 묻지 않으며 카드 리스트 출력도 생략한다 — 보고서가 곧 결과물이다.

### 5단계 — 최종 docx 1개 생성 (제1부 뉴스 + 제2부 과제)

**docx 는 한 번만, 단일 2부 보고서로 만든다.** 뉴스만 담은 별도 docx 는 생성하지 않는다.

```bash
SKILL_DIR="<이 스킬의 디렉토리 경로>"
REPO_ROOT="<리포 루트>"
DIR="$REPO_ROOT/outputs/fa-task-discovery"
D=$(date +%Y%m%d)
cd "$SKILL_DIR/scripts"
npm install --silent   # 최초 1회 또는 node_modules 없을 때만

node generate_report.js "$DIR/FA_기술과제후보_$D.docx" "$DIR/tasks_$D.json" "$DIR/news_$D.json"
```

생성 후:
- 파일명: `FA_기술과제후보_{YYYYMMDD}.docx` (제1부 뉴스 + 제2부 과제)
- `computer://` 링크로 사용자에게 제공

### 6단계 — 1회 commit·push + 단일 PR 자동 생성

⚠️ **이 스킬은 한 실행에서 PR 을 단 1개만 만든다.** 1·2·3·4·5단계 도중에는 절대 git/PR 을 호출하지 않고, 5단계의 최종 docx 까지 완성된 다음에 한 번에 commit·push·PR 을 수행한다.

```bash
cd "$REPO_ROOT"
D=$(date +%Y%m%d)
BR="fa-task-discovery/auto/$D"
git checkout -B "$BR"
# news.json + tasks.json + 최종 docx 를 한 커밋에 묶어 push (중간 push 없음)
git add "outputs/fa-task-discovery/news_$D.json" \
        "outputs/fa-task-discovery/tasks_$D.json" \
        "outputs/fa-task-discovery/FA_기술과제후보_$D.docx"
git commit -m "FA 기술과제후보 $D (뉴스+과제 2부)"
# 네트워크 실패 시 지수 백오프(2/4/8/16s)로 최대 4회 재시도
git push -u origin "$BR"
```

그 다음 GitHub MCP 로 PR 을 1개 생성한다:
- 도구: `mcp__github__create_pull_request` (`gh` CLI 사용 금지)
- `base` = 기본 브랜치(보통 `main`), `head` = `$BR`
- `title` = `FA 기술과제후보 <YYYYMMDD>`, `body` = 뉴스 건수·과제 건수·핵심 항목 요약
- **같은 `head` 의 열린 PR 이 있으면 새로 만들지 말고 그대로 둔다** (`mcp__github__list_pull_requests` 로 선확인). 같은 날 재실행이면 동일 브랜치에 추가 커밋이 쌓여 기존 PR 이 자동 갱신된다.
- PR 은 **draft 가 아닌 일반 PR** 로 생성 (산출물을 빠르게 메인까지 반영).

⚠️ `git push origin main` 등 기본 브랜치 직접 푸시는 절대 하지 않는다 — 반드시 PR 경유.
⚠️ `references/INDEX.md`·`references/manifest.json` 은 `.gitignore` 처리되어 자동으로 PR 에 포함되지 않는다 — 추적·커밋 시도하지 말 것.

### 7단계 — 카카오톡 요약 전송 (단일 메시지)

`KakaotalkChat-MemoChat` 도구로 나에게 채팅 전송. **반드시 200자 이상**, 핵심 기사 URL 2건 이상 + PR/docx 경로 포함. (도구 설명에 "최대 200자"라 적혀 있어도 실제로는 더 긴 메시지가 정상 전송되므로 분량을 충분히 채운다.) 골격:

```
🏭 FA 기술과제 브리핑 — YYYY.MM.DD

🚗 AMR/AGV: [핵심 동향 1줄]
🤝 협동로봇: [핵심 동향 1줄]
🌐 디지털트윈: [핵심 동향 1줄]
📦 스마트물류: [핵심 동향 1줄]
🤖 AI/피지컬AI: [핵심 동향 1줄]

📰 최근 1개월 뉴스 M건 / 📋 과제 후보 N건 도출
주요 후보:
• [과제명 1]
• [과제명 2]
• [과제명 3]
🔗 핵심 기사: [기사1] [URL1] · [기사2] [URL2] · [기사3] [URL3]
※ 상세 보고서 → outputs/fa-task-discovery/FA_기술과제후보_YYYYMMDD.docx (제1부 뉴스+제2부 과제, PR: [PR URL])
```

## 실행 모드

이 스킬은 **단일 모드(자동·1실행=1PR)** 만 지원한다. 트리거(수동/스케줄/명시 요청) 무관하게 1~7단계를 한 번에 끝까지 수행한다. 1·2·3·4·5단계는 commit·push·PR을 호출하지 않고, **6단계 단 1회만** 날짜 브랜치 커밋·푸시 + 메인 대상 PR 자동 생성을 수행한다. 기본 브랜치 직접 push 금지.

## 실패 대응

- 서브에이전트 1개 실패 → 나머지로 부분 보고서, 실패 명시
- 최근 1개월 뉴스 0건인 분야 → 제1부에 "이번 주기 신규 동향 없음" 자동 표기 (스크립트 처리)
- 오늘자 news.json 손상/빈 배열 → 1·2단계를 재실행해 다시 만든 뒤 3단계로 계속 (실행 중단 금지)
- 과제 후보 5개 미만 → "이번 주기 신규 동향 부족" 명시
- docx 스크립트 실패 → `pip install python-docx` 후 Python 폴백 시도
- 카카오톡 실패 → docx 링크만 제공

## 결과물 체크리스트 (1실행에서 모두 충족)

- [ ] 6개 분야 모두 탐색, 모든 뉴스 최근 30일 이내 + 출처 URL 포함
- [ ] `news_{today}.json` + `tasks_{today}.json` + `FA_기술과제후보_{today}.docx`(2부) 모두 생성
