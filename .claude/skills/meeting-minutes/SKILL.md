# 회의록 생성 스킬

녹취록 → LGES 표준 회의록 PPTX 자동 생성 워크플로우.

## 전체 흐름 요약

```
1단계: 녹취록 입력 확인
2단계: 참석자 정보 수집 (인터랙티브 UI)
3단계: AI가 회의 내용 전체 요약 생성
4단계: 사용자가 포함할 요약 항목 선택 + 수정 (인터랙티브 UI)
5단계: 최종 플랜 확인 후 PPTX 일괄 생성 (파일명 + 내용)
```

---

## 1단계: 녹취록 입력 확인

- 사용자가 파일을 업로드했으면 `/mnt/user-data/uploads/`에서 읽는다.
- 텍스트를 직접 붙여넣었으면 그것을 사용한다.
- 녹취록이 없으면: "녹취록 파일을 업로드하거나 내용을 붙여넣어 주세요."라고 안내한다.

녹취록에서 다음 정보를 자동 파악한다:
- 회의 날짜/시간 (파일명이나 텍스트 첫 줄에서 추출)
- 발화자 목록 (참석자 1, 참석자 2 등 → 실제 이름으로 매핑 필요)
- 회의 주제 (내용에서 추론)

---

## 2단계: 참석자 정보 수집

**`scripts/attendee_ui_template.html`을 읽어서 show_widget으로 렌더링한다.**

렌더링 전처리:
1. `scripts/attendee_ui_template.html` 파일을 읽는다
2. `ATTENDEE_ROWS_PLACEHOLDER`를 감지된 발화자 수에 맞는 JSON 배열로 교체한다
   - 예: 발화자 3명이면 `[["LGES","","",""],["LGES","","",""],["LGES","",""]]`
   - 녹취록에서 실명을 감지했으면 name 필드에 자동 채워준다
3. show_widget 호출 시 `widget_code`에 교체된 HTML을 그대로 넣는다

아래 정보를 행 단위로 입력받는다:

| 필드 | 설명 | 예시 |
|------|------|------|
| 회사/법인 | 소속 회사 | LGES, ESMI |
| 조직 | 소속 팀·담당 | FA기술담당 |
| 직책 | 직위 | 책임, 팀장, 부장 |
| 이름 | 성명 | 강모원 |

- 녹취록에서 발화자(참석자 1, 참석자 2...)가 확인되면 그 수만큼 행을 미리 생성해 준다.
- 행 추가/삭제 가능
- "다음 단계" 버튼 클릭 시 `sendPrompt()`로 정보 전달

**UI 출력 형식 예시 (sendPrompt로 전달):**
```
[참석자정보]
LGES|FA기술담당|책임|강모원
LGES|FA기술담당|팀장|최상훈
LGES|FA기술담당|팀장|육준호
```

---

## 3단계: AI 회의 내용 요약 생성

참석자 정보를 받은 후, 녹취록 전체를 분석하여 **6~10개의 핵심 요약 항목**을 생성한다.

각 요약 항목 형식:
```
{
  "id": 1,
  "주관부서": "FA기술담당",
  "요약": "ESS 라인 3개 배치 케이스 2가지 검토 - 기존 모듈 라인 영역 vs 신규 빈 부지",
  "코멘트": "10월 16일 SOP 일정 기준으로 검토 중",
  "기한": "25.10"
}
```

요약 생성 원칙:
- 한 항목당 한 가지 결정사항 또는 논의사항만 담는다
- 액션 아이템이 있으면 별도 항목으로 분리한다
- 중복되거나 너무 세부적인 내용은 통합한다
- "주관부서"는 해당 안건을 주도한 팀 또는 기본값(요청자 소속팀)으로 채운다

---

## 4단계: 요약 선택 및 수정 UI

**`scripts/summary_ui_template.html`을 읽어서 show_widget으로 렌더링한다.**

렌더링 전처리:
1. `scripts/summary_ui_template.html` 파일을 읽는다
2. `SUMMARY_ITEMS_PLACEHOLDER`를 3단계 요약 JSON 배열로 교체한다
   ```json
   [{"id":1,"주관부서":"FA기술담당","요약":"...","코멘트":"...","기한":"25.10"}, ...]
   ```
3. show_widget 호출 시 `widget_code`에 교체된 HTML을 넣는다

### UI 구성

**상단: 전체 요약 목록**
- 각 요약 항목을 카드 형태로 표시
- 체크박스로 포함/제외 선택 (기본값: 전체 선택)
- 항목 순서 위아래 이동 가능 (드래그 또는 버튼)

**각 카드 내부:**
- 주관부서 (수정 가능)
- 요약 내용 (수정 가능 텍스트 영역)
- 코멘트 (수정 가능)
- 기한 YY.MM (수정 가능)
- ✏️ 수정 버튼 → 인라인 편집 활성화
- 🗑️ 삭제 버튼

**하단 버튼:**
- `+ 항목 추가` → 빈 카드 생성
- `최종 확인` → sendPrompt로 선택된 항목 전달

**sendPrompt 전달 형식:**
```
[선택요약]
1|FA기술담당|ESS 라인 3개 배치 케이스 검토|10월 16일 SOP 기준|25.10
2|FA기술담당|물류 자동화 방식 결정 필요|케이스별 물류 차이 존재|25.10
...
```

> **완료/보고 기한 셀 규칙**: 기한(YY.MM) 또는 짧은 Comment 중 하나만 표시한다.
> 기한이 있으면 기한 우선, 없으면 Comment 앞 15자. 두 값을 동시에 표시하지 않는다.

## 5단계: 플랜 확인 및 PPTX 생성

### 5-1. 플랜 출력

사용자에게 텍스트로 플랜을 보여준다:

```
📋 회의록 생성 플랜

파일명: 260423_LGES_FA기술담당_ESS라인배치검토_회의록_v1.pptx

슬라이드 내용:
├── 일시/장소: 2026년 04월 23일(목) 20:57, [장소]
├── 참석자:
│   [LGES] FA기술담당 책임 강모원, 팀장 최상훈, 팀장 육준호
├── 요약 항목 (6개):
│   1. ESS 라인 3개 배치 케이스 검토
│   2. 물류 자동화 방식 결정 필요
│   ...

생성하시겠습니까? (확인/수정)
```

파일명 규칙: `YYMMDD_회사_조직_회의제목_회의록_v1.pptx`
- YYMMDD: 회의 날짜
- 회사: 주최 회사명
- 조직: 주최 조직명  
- 회의제목: 회의 내용 핵심 키워드 (최대 15자, 공백 없이)

### 5-2. PPTX 생성

사용자가 확인하면 `scripts/generate_minutes.py`를 실행한다:

```bash
python3 scripts/generate_minutes.py \
  --template assets/template.pptx \
  --output "/mnt/user-data/outputs/<파일명>" \
  --meta '{"date":"26.04.23","weekday":"목","time":"20:57~","location":"온라인","dept":"FA기술담당"}' \
  --attendees 'LGES|FA기술담당|책임|강모원\nLGES|FA기술담당|팀장|최상훈' \
  --items '1|FA기술담당|내용1|코멘트1|25.10\n2|FA기술담당|내용2|코멘트2|25.11'
```

생성 후 `extract-text`로 내용 검증한 뒤 `present_files`로 파일을 사용자에게 전달한다.

## 오류 처리

| 상황 | 대응 |
|------|------|
| 날짜 파악 불가 | 사용자에게 직접 입력 요청 |
| 요약 항목이 6개 미만 | 남은 슬롯을 빈 항목으로 채움 |
| 요약 항목이 6개 초과 | 선택 UI에서 사용자가 제외 선택 |
| 장소 정보 없음 | "[장소]" 플레이스홀더 유지 |
