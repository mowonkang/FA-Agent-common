# LGES PPT 작업 가이드 (에이전트 기본 룰)

이 리포의 에이전트가 `.pptx` 산출물을 만들 때 따르는 **기본 작업 가이드**.
`templates/사내양식/LGES_양식_톤_지침.md` 가 양식 분석 기반의 톤 사전이라면, 본 문서는 그 위에서 **에이전트가 실제로 슬라이드를 생성할 때 적용해야 할 단순화·통일 규칙**이다.

대표 적용 예: `scripts/build_flexibility_summary.py` →
`outputs/유연성지표_요약_2026-05-14.pptx`.

---

## 0. 본 가이드의 우선순위

- 본 가이드 ↔ `LGES_양식_톤_지침.md` 가 충돌하면 **본 가이드를 우선** 따른다.
- 본 가이드는 임원 보고용 단일 페이지 슬라이드(요약·결정 지원) 생성에 최적화된 단순화 규칙이다.
- **본 가이드의 톤·색·사이즈·레이아웃은 권장 기준(참고)이다.** 자료 구성·가독성·스토리 향상을 위해 자율 조정할 수 있다. 단 ① 텍스트가 박스를 벗어나면 안 되고 ② 임원 보고 즉시 활용 품질·짜임새·스토리가 최우선이다(양식 모방보다 메시지 전달 우선).

---

## 0-A. 문서 구조 원칙 (메인 + 보조 슬라이드)

PPT 산출물은 **메인 1~2장 + 보조 N장** 구조를 기본으로 한다.

### 메인 슬라이드 (1~2장)
- 임원이 한눈에 의사결정 가능한 **요약 슬라이드**.
- 1장이 원칙이지만 주제가 클 때는 최대 2장까지 허용 (예: 1장 정의·산식, 2장 현황·결정 포인트).
- 칸 안에 내용이 **반드시 모두 들어가야 함** — 한 글자라도 박스를 벗어나면 메인 자격 없음.
- 내용이 넘치면 **보조 슬라이드로 분리**하고 메인은 요약만 남긴다.

### 보조 슬라이드 (N장)
- 메인에 다 담지 못한 상세를 **여러 장으로 풀어서** 첨부.
- 회색조 톤(흰색 + 회색)으로 메인과 시각적으로 구분 (강조 파랑은 메인 한정 권장, 보조는 회색·검정 위주).
- 보조 슬라이드도 사이즈·폰트·풋노트 규칙은 동일하게 따른다.
- 보조 슬라이드 첫 줄에 `[보조 N/M] 주제` 식 라벨 띠로 메인과 구분.

### 분리 기준 (메인 → 보조로 넘길 때)
- 텍스트가 박스 안에서 잘리거나 줄바꿈이 깨질 때
- bullet 이 한 박스 안에 7~8 줄을 넘어갈 때
- 표 행이 8행 이상으로 커질 때 (요약 표만 메인, 상세 표는 보조로)
- 그래프·이미지·도형 다수가 필요할 때 (메인은 가능한 텍스트 중심으로)

### 권장 흐름
```
메인 1장 : 한 장 요약 (4사분면) — 정의·기준·핵심 데이터·결정 포인트
   └─ 보조 1 : 정의·산식 상세 (분자/분모 풀어쓰기, 산정 예외)
   └─ 보조 2 : 사례 데이터 상세 표 (메인 표보다 행·열 많음)
   └─ 보조 3 : 현황 매핑 상세 (KPI·로드맵 과제별 진척)
   └─ 보조 4 : 결정 사항 옵션 비교 / 부록
```

## 1. 색 팔레트 (가장 중요)

### 1-1. 기본 색

| 용도 | HEX | RGB | 설명 |
|---|---|---|---|
| 배경 | `#FFFFFF` | 255,255,255 | 화이트 |
| **본문 텍스트** | `#000000` | 0,0,0 | **검정.** 모든 텍스트 색은 검정 또는 회색 (그 외 색 금지) |
| 카드 배경 (옵션) | `#F5F5F5` | 245,245,245 | LIGHT_GRAY |
| 표 헤더 배경 | `#C0C0C0` | 192,192,192 | GRAY |
| 표 강조 행 / 산식 박스 | `#E5E5E5` | 229,229,229 | SOFT_GRAY |
| 카드 테두리 | `#8C8C8C` | 140,140,140 | MID_GRAY |
| 보조 텍스트 | `#8C8C8C` | 140,140,140 | MID_GRAY (예시·각주 보조 설명) |
| 라벨 띠 배경 | `#4D4D4D` | 77,77,77 | DIM_GRAY (라벨 띠는 진회색 + 흰 글씨) |
| 점잖은 강조용 차콜 | `#2A2A2A` | 42,42,42 | CHARCOAL (**최소 사용**. 일반 중요 포인트는 차콜 대신 그냥 검정 Bold 로) |

### 1-2. 강조색 (포인트만)

| 용도 | HEX | RGB | 설명 |
|---|---|---|---|
| **강조 (특이 포인트만)** | `#0000FF` | **R0 G0 B255** | **순수 파랑.** 슬라이드 1장당 강조는 평균 **3~5 개 포인트** 이내. |
| 강조 박스 배경 | `#E6ECFF` | 230,236,255 | 옅은 파랑. 핵심 메시지·임원 결정 박스 같은 **단 1~2 개** 박스에만. |

**파랑 사용 규칙**

- 슬라이드의 정말 **특이한 부분**(=한 눈에 들어와야 할 결론·지표·결정 포인트)에만 사용.
- 본문 bullet, 일반 섹션 헤더, 표의 모든 행에 무차별 적용 **금지**.
- 권장 적용 위치:
  - 슬라이드 핵심 메시지 박스 (상단)
  - 산식·정의의 핵심 수식
  - "임원 결정 포인트" 같은 결론 박스
  - 표에서 강조해야 할 단 1행 (예: Total, 유연형 카테고리)
- 일반 섹션 헤더(`[적용 방식]`, `[예외 처리]` 등) → **검정 Bold** 로만 (색 바꾸지 않음).
- 일반적인 중요 포인트 = **Bold 만**, 색은 검정 유지. 색 변경은 파랑·차콜·회색 보조설명에만.

### 1-3. 풋노트 전용 — 초록곰팡이

| 용도 | HEX | RGB | 설명 |
|---|---|---|---|
| 풋노트 / 출처 | `#006600` | **G102** (R0 G102 B0) | "초록곰팡이". **풋노트에만** 사용. |

**풋노트 규칙**

- 풋노트는 반드시 **별도 텍스트 박스** 로 슬라이드 하단(y ≈ 18.3cm)에 배치.
- 색은 `#006600` (R0 G102 B0), **Bold**, 사이즈 8pt.
- 폰트는 본문과 동일(`set_run` 헬퍼가 한·영 자동 분리).
- 본 풋노트 외 다른 곳에는 초록 사용 금지.

### 1-4. 절대 금지

- 무지개 색·오렌지·빨강·노랑 강조색 일절 금지.
- 강조색을 본문 다수에 사용 (블루 침투 금지).
- 그림자·그라데이션·블러 효과.

## 2. 폰트 (한·영 자동 분리)

### 2-1. 폰트 패밀리

| 분류 | 폰트 |
|---|---|
| 한글 (ea) | **LG스마트체 Regular** |
| 영문·숫자 (latin / cs) | **Arial Narrow** |

한 run 안에 한글+영문이 섞여도 PowerPoint 가 자동 분리하도록 **반드시 ea / latin 모두 지정** 해야 한다. `scripts/build_flexibility_summary.py` 의 `set_run` 헬퍼가 이를 처리한다.

```python
def set_run(run, text, *, font=FONT_BODY, size=11, bold=False, color=BLACK):
    """ea(한글)=LG스마트체, latin(영문)=Arial Narrow 자동 분리."""
    ea_font, latin_font = FONT_BODY, FONT_EMPH
    rPr = run._r.get_or_add_rPr()
    for tag, fnt in (("ea", ea_font), ("cs", ea_font), ("latin", latin_font)):
        el = etree.SubElement(rPr, qn(f"a:{tag}"))
        el.set("typeface", fnt)
```

### 2-2. 폰트 사이즈 스케일 (6단)

직접 `size=` 숫자를 흩어 적지 **말고** 아래 상수만 사용.

```python
SZ_TITLE   = 16   # 슬라이드 타이틀
SZ_BAND    = 12   # 사분면·섹션 라벨 띠 (헤더)
SZ_SECTION = 11   # 카드 내 섹션 헤더 ([적용 방식] 등)
SZ_BODY    = 10   # 본문 bullet, 표 셀, 책임자 표기
SZ_SUB     = 9    # 보조 설명·예시·시뮬레이션
SZ_FOOT    = 8    # 풋노트 / 출처
```

**한 슬라이드에 4~5 개 이내** 의 사이즈만 사용. 즉흥적으로 13pt·14pt 같은 중간값 쓰지 않는다.

## 3. 슬라이드 레이아웃

### 3-1. 슬라이드 사이즈

```python
prs.slide_width  = Cm(27.52)
prs.slide_height = Cm(19.05)
```

LGES 그룹 표준 (16:9 와이드 아님).

### 3-2. 단일 페이지 4사분면 표준 레이아웃 (메인 슬라이드 권장 형태)

임원 보고용 메인 1장 요약은 아래 4사분면 구조를 권장. **모든 내용이 박스 안에 들어가야 한다** — 넘치면 보조 슬라이드로 빼고 메인은 요약만 남긴다.

```
┌───────────────────────────── 타이틀 (16pt B) ───── 우상단: 책임자 (10pt B 파랑) ──┐
├──────────────────────────────────────────────────────────────────────────────────┤
│           [핵심 메시지] 박스 (LIGHT_BLUE 배경 + 파랑 보더, 본문 10pt)             │
├────────────────────────────────┬─────────────────────────────────────────────────┤
│ ① 라벨 띠 (DIM_GRAY + 흰글)    │ ② 라벨 띠 (DIM_GRAY + 흰글)                    │
│                                │                                                 │
│  카드 내용 (MID_GRAY 보더)     │  카드 내용                                      │
├────────────────────────────────┼─────────────────────────────────────────────────┤
│ ③ 라벨 띠                      │ ④ 라벨 띠                                       │
│                                │                                                 │
│  카드 내용                     │  카드 내용 + [결정 포인트] 박스 (파랑 강조)     │
├──────────────────────────────────────────────────────────────────────────────────┤
│  * 출처 : ...  ← 별도 텍스트박스, 초록곰팡이(G102) 8pt Bold                       │
└──────────────────────────────────────────────────────────────────────────────────┘
```

좌표 권장값 (참고):

| 영역 | y 시작 | 높이 | 비고 |
|---|---|---|---|
| 타이틀 | 0.25 | 1.0 | 16pt B |
| 구분선 | 1.35 | - | MID_GRAY 0.75pt |
| 핵심 메시지 박스 | 1.55 | 1.95 | LIGHT_BLUE + 파랑 보더 |
| 상단 사분면 (Q1·Q2) | 3.75 | 7.20 | 라벨 띠 0.7 + 카드 6.5 |
| 하단 사분면 (Q3·Q4) | 11.10 | 6.90 | 라벨 띠 0.7 + 카드 6.2 |
| 풋노트 | 18.30 | 0.55 | 별도 텍스트박스, 초록곰팡이 |

### 3-3. 라벨 띠 (사분면 헤더)

- 배경: `DIM_GRAY (#4D4D4D)`, 글자: `WHITE (#FFFFFF) 12pt B`
- 형식: `"① 정의 및 산식"`, `"② 고정형 vs 유연형 분류 기준"`
- 카드 외곽선은 별도 `add_rect(..., line=MID_GRAY, line_w=0.75)`

### 3-4. 카드 내 섹션 헤더

- 형식: `[적용 방식]`, `[예외 처리]`, `[기준 변경 논의 사항]`
- **검정(#000000) Bold 11pt**. 차콜·파랑 사용 **금지** (특이 포인트가 아님)
- 일반적인 중요 포인트는 색을 바꾸지 말고 **Bold 만** 적용

### 3-5. 강조 박스 (옅은 파랑 배경 + 회색 외곽)

슬라이드당 **0~2 개** 만:
- 상단 "핵심 메시지" 박스
- 하단 "임원 결정 포인트" 박스
- 산식·지표 같은 핵심 정의 박스

**박스 외곽선(line)은 회색·검정만**. 배경은 옅은 파랑 가능, 단 외곽은 회색 유지.

```python
add_rect(left, top, w, h, fill=LIGHT_BLUE, line=MID_GRAY, line_w=0.75)
```

### 3-6. 표

- 헤더 행: `GRAY (#C0C0C0)` 배경, 10pt Bold 검정, 가운데 정렬
- 데이터 행: 흰색, 10pt 검정, 본문 좌측·숫자 가운데
- 합계·Total 행: `SOFT_GRAY (#E5E5E5)` 배경, **수치만 파랑 Bold** (한 행 한정)
- 첫 컬럼이 카테고리 라벨이면 Bold (색은 검정 또는 차콜)
- 메인 표는 최대 6~7 행 권장. 더 길어지면 보조 슬라이드의 상세 표로 분리.

### 3-7. 보조 슬라이드 레이아웃

- 라벨 띠 첫 줄에 `[보조 N/M] 주제` 표기 (DIM_GRAY 배경 + 흰글)
- 강조 파랑은 가능한 사용하지 않고 회색·검정으로 마무리
- 본문 폰트·사이즈·풋노트 규칙은 메인과 동일
- 한 보조 슬라이드는 **하나의 주제**만 다룬다 (예: 정의 상세, 사례 데이터, 매핑 상세 등)
- 슬라이드 우상단에 책임자 표기 유지

## 5. 헬퍼 함수 표준 (재사용 권장)

`scripts/build_flexibility_summary.py` 의 다음 헬퍼를 신규 PPT 빌더에 그대로 가져와 쓸 것:

| 헬퍼 | 역할 |
|---|---|
| `set_run(run, text, ...)` | run 1개에 한·영 분리 폰트·사이즈·색 적용 |

| `add_text(slide, l, t, w, h, lines, ...)` | 텍스트 박스 + 본문 |
| `add_label_band(slide, l, t, w, text)` | 사분면 라벨 띠 (DIM_GRAY + 흰글) |

- [ ] 메인에 다 못 담은 내용은 **보조 슬라이드 N장**으로 분리

- [ ] 텍스트 색은 검정 또는 회색 계열만 (그 외 색은 파랑·풋노트 초록만)
- [ ] 모든 박스·카드 외곽선은 **검정 또는 회색만** (MID_GRAY / LINE_GRAY / BLACK)
- [ ] 라벨 띠 DIM_GRAY 배경 + 흰글, 파랑 외곽선은 사용하지 않음
- [ ] 박스 안 내용이 카드 영역을 벗어나지 않음 (y 좌표 + height ≤ 카드 끝)

## 8. 임원 보고 양식 카탈로그 & 적용 예시

`templates/사내양식/(보고양식)YYMMDD_보고제목_v1.0.pptx` (임원 보고용 12종
슬라이드 양식 모음) 를 8개 재사용 보고 블록으로 정리한 카탈로그가 있다.
상위 보고 요청 시 카탈로그에서 블록을 골라 **메인(블록 A) + 보조(B~H)**
구조로 조립한다.

| 블록 | 이름 | 용도 |
|---|---|---|
| A | 보고 프레임 표준 | 메인 한 장 요약 레이아웃 |
| B | Master Plan 일정·로드맵 | 분기 Gantt, 추진 일정 |
| C | 투자비·예산 (Capex/KPI) | 투자비 As-is/To-be, KPI 진척 |
| D | 추진체계·조직도 | ES/PRI/CNS 매트릭스 |
| E | 전략 추진방향·컨셉 | 지향점·중점과제, 컨셉 |
| F | 기술 사양·구성 최적화 | 구성 방안, 사양·경량화 |
| G | As-Is / To-Be 전환 | 프로세스·시스템 전환 비교 |
| H | 향후계획·검증 | 보조 과제, PoC 검증 로드맵 |

- 카탈로그: [`보고양식_카탈로그.md`](./보고양식_카탈로그.md) (사람용) /
  `보고양식_카탈로그.json` (판독용, `python3 scripts/build_report_catalog.py`)
- 적용 예시: `scripts/build_fa_exec_report.py` →
  `outputs/FA자동화_임원보고_예시_2026-05-18.pptx`
  (메인 1 + 보조 4, 스토리 6단계, references 출처 인용)
- 카탈로그의 양식·톤은 **참고**이며 가독성·스토리 위해 자율 조정 가능.
  단 무오버플로·임원 즉시 활용 품질은 필수.
