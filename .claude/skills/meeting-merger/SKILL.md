# Meeting Merger — 녹취록 + OneNote → Word 문서 생성 스킬

## 목적

두 소스(녹취록, OneNote)를 받아 **단 하나의 정제된 회의록 .docx**를 생성한다.

---

## 입력 형태

| 소스 | 역할 | 특징 |
|------|------|------|
| OneNote 메모 | **기준 구조** | 담당자가 미팅 중 직접 작성. 구조는 있으나 오탈자·누락 가능 |
| 녹취록 | **보완 소스** | 발화 그대로 기록. 내용은 풍부하나 구어체·중복 많음 |

---

## 처리 단계 (반드시 순서대로 실행)

### Step 1. 소스 분석

두 소스를 각각 읽어 아래 항목을 추출한다:

**OneNote에서 추출:**
- 전체 계층 구조 (헤더, 팀/사이트, 항목)
- 날짜, 참석자, 안건 목록
- 기존에 작성된 액션아이템·결정사항

**녹취록에서 추출:**
- OneNote에 없는 추가 발언·맥락·수치
- 결정된 사항 ("~로 결정", "~하기로 함")
- 액션아이템 ("~해주세요", "~담당해주십시오", "~검토 부탁")
- 이슈·리스크 언급 ("문제", "리스크", "지연", "차질", "Risk")
- 일정 언급 ("~월 ~일", "차주", "금주", "다음 주")

### Step 2. 병합 전략

- **구조**: OneNote 계층을 그대로 유지
- **내용 보완**: 녹취록에만 있는 중요 내용 → 해당 섹션에 삽입
- **중복 제거**: 동일 내용이 양쪽에 있으면 OneNote 문구 기준, 녹취록으로 표현 보완
- **구어체 변환**: 녹취록 발화체 ("~거든요", "~잖아요") → 문어체 문장으로 변환
- **오탈자 교정**: 두 소스 모두에서 명백한 오탈자·띄어쓰기 오류 수정

### Step 3. 섹션별 정리

아래 7개 섹션을 순서대로 구성한다:

```
[1] 회의 기본 정보        — 날짜, 참석자, 장소, 목적
[2] 회의 요약             — 핵심 내용 3~5줄 (전체를 읽지 않아도 파악 가능하게)
[3] 결정 사항             — Decision 목록 (표 형식)
[4] 액션 아이템           — 담당자·기한 포함 체크리스트 (표 형식)
[5] 이슈 / 리스크         — 식별된 문제·리스크 목록 (표 형식)
[6] 일정표                — 날짜별 일정 정리 (표 형식)
[7] 전체 상세 내용        — OneNote 구조 기반 섹션별 상세 기록
```

### Step 4. Word 문서 생성

docx 스킬 기준에 따라 JavaScript(docx-js)로 생성.

**반드시 아래 명령어를 먼저 실행:**
```bash
npm install -g docx
```

**페이지 설정:**
- 용지: A4 (11906 × 16838 DXA)
- 여백: 상하좌우 1134 DXA (약 2cm)
- 기본 폰트: 맑은 고딕 (한국어 문서), fallback Arial

**색상 팔레트 (프리미엄 네이비 + 골드 포인트):**

| 용도 | HEX |
|------|-----|
| 메인 헤더 배경 | `1C2B3A` (딥 네이비) |
| 섹션 구분선 / 서브헤더 | `2D4159` (슬레이트) |
| 골드 포인트 (섹션 레이블, 강조) | `B8883A` (웜 골드) |
| 결정사항 행 배경 | `EBF0F8` / `E0EAF5` (교번) |
| 액션아이템 행 배경 | `EAF4EC` / `DFF0DF` (교번) |
| 이슈/리스크 행 배경 | `FAEEE9` / `F5DDD8` (교번) |
| 일정 행 배경 | `F5F0E8` / `EDE7D5` (교번) |
| 표 셀 테두리 | `D8E2EE` (소프트) |
| 상태 — 완료 | `DFF0D8` (녹) |
| 상태 — 진행중 | `D6EAF8` (청) |
| 상태 — 예정 | `F0E6CF` (골드) |
| 심각도 — 높음 | `FADBD8` (적) |
| 심각도 — 중간 | `FDEBD0` (주황) |
| 심각도 — 낮음 | `D5F5E3` (녹) |

**헤더/푸터:**
- 헤더: 문서 제목 + 날짜 (우측 정렬)
- 푸터: 페이지 번호 (중앙)

## 섹션별 상세 출력 형식

### [1] 회의 기본 정보

```
회의명:   2604_W03 FA기술담당 주간미팅
일시:     2026년 4월 20일 (월) 08:17
장소:     (녹취록/OneNote에서 추출, 없으면 생략)
참석자:   (추출된 인원, 없으면 생략)
작성자:   (없으면 생략)
```

### [2] 회의 요약

핵심 내용 3~5줄. 불릿 형식. 전체를 안 읽어도 이 섹션만으로 미팅 결과를 파악할 수 있어야 함.

### [3] 결정 사항 표

| # | 결정 내용 | 관련 팀/사이트 | 비고 |
|---|-----------|----------------|------|
| 1 | ...        | ESGM2          | ...  |

### [4] 액션 아이템 표

| # | 할 일 | 담당자 | 기한 | 상태 |
|---|-------|--------|------|------|
| 1 | ...   | 김영곤 | 5/3  | 진행중 |

상태값: `완료` / `진행중` / `예정` / `보류`
녹취록·OneNote에서 "완료" 명시된 경우만 `완료` 처리.

### [5] 이슈 / 리스크 표

| # | 이슈 내용 | 심각도 | 관련 팀 | 조치 방향 |
|---|-----------|--------|---------|-----------|
| 1 | ...        | 높음   | ESGM2   | ...        |

심각도: `높음` / `중간` / `낮음`
판단 기준: 생산 차질·일정 지연 → 높음, 검토 필요 → 중간, 모니터링 → 낮음

### [6] 일정표

| 날짜 | 시간 | 구분 | 내용 |
|------|------|------|------|
| 4/20 | 09:00 | [센터] | 인프라업무보고 |

### [7] 전체 상세 내용

OneNote 구조를 헤더 계층으로 변환. 녹취록 보완 내용은 이탤릭체로 표시하여 구분.
```
[녹취록 보완] 내용...
```

## 오탈자 교정 기준

교정 O:
- 명백한 띄어쓰기 오류 (예: "완료보고" → "완료 보고")
- 자음/모음 오류 (예: "미탱" → "미팅")
- 숫자·단위 오류 (예: "5워" → "5월")

교정 X:
- 약어·사내 용어 (AMR, OHT, CPO, ESOC 등)
- 담당자명·사이트 코드
- 의미가 불분명하여 판단이 어려운 경우 → 원문 유지 + 문서 끝에 "교정 불확실 항목" 섹션 추가

---

## 출력 파일 규칙

- 저장 경로: `/mnt/user-data/outputs/meeting_YYYYMMDD.docx`
- 날짜는 미팅 날짜 기준 (없으면 오늘 날짜)
- 생성 후 반드시 `python scripts/office/validate.py` 로 검증

---

## 주의사항

- 구어체 발화 ("어, 그니까", "저기", 필러 단어)는 변환 시 제거
- 동일 내용 반복 발화는 한 번만 기록
- 발화자 특정이 어려운 경우 발화자 표기 생략
- 전체 상세 내용 섹션에서 OneNote와 녹취록 보완 내용의 출처를 구분하여 가독성 확보
