# LGES 사내 브랜드 가이드 스킬

본 스킬은 본 리포의 **`templates/사내양식/LGES_PPT_작업_가이드.md`** 와 기준 구현 **`scripts/build_flexibility_summary.py`** 를 SKILL 형태로 캡슐화한 것이다. 모든 사내 보고용 산출물 빌더는 이 룰을 우선 따른다.

## 0. 우선순위

본 스킬 ↔ `templates/사내양식/LGES_양식_톤_지침.md` 가 충돌하면 **본 스킬을 우선** 한다. 본 스킬의 톤·색·사이즈·레이아웃은 권장 기준이며, 자료 구성·가독성·스토리를 위해 자율 조정할 수 있다. 단 ① 텍스트가 박스를 벗어나면 안 되고 ② 임원 보고 즉시 활용 품질·스토리가 최우선이다.

## 1. 색 팔레트 (가장 중요)

### 1-1. 기본 색 (모든 산출물 공통)

| 용도 | HEX | RGB | 변수명 |
|---|---|---|---|
| 배경 | `#FFFFFF` | 255,255,255 | WHITE |
| **본문 텍스트** | `#000000` | 0,0,0 | **BLACK** (모든 텍스트 색은 검정 또는 회색) |
| 카드 배경 (옵션) | `#F5F5F5` | 245,245,245 | LIGHT_GRAY |
| 표 헤더 배경 | `#C0C0C0` | 192,192,192 | GRAY |
| 표 강조 행 / 산식 박스 | `#E5E5E5` | 229,229,229 | SOFT_GRAY |
| 카드 테두리 | `#8C8C8C` | 140,140,140 | MID_GRAY |
| 보조 텍스트 (예시·각주) | `#8C8C8C` | 140,140,140 | MID_GRAY |
| 라벨 띠 배경 (사분면 헤더) | `#4D4D4D` | 77,77,77 | DIM_GRAY |
| 점잖은 강조용 차콜 (최소 사용) | `#2A2A2A` | 42,42,42 | CHARCOAL |

### 1-2. 강조색 — 슬라이드당 **3~5 개 이내**만

| 용도 | HEX | RGB | 변수명 |
|---|---|---|---|
| **강조 (특이 포인트만)** | `#0000FF` | **R0 G0 B255** | **BLUE** (순수 파랑) |
| 강조 박스 배경 | `#E6ECFF` | 230,236,255 | LIGHT_BLUE (1~2 개 박스만) |

**파랑 사용 규칙**

- 슬라이드의 **특이한 부분**(한 눈에 들어와야 할 결론·지표·결정 포인트)에만 사용
- 일반 섹션 헤더(`[적용 방식]`, `[예외 처리]` 등) → **검정 Bold** 로만 (색 바꾸지 않음)
- 일반 중요 포인트 = **Bold 만**, 색은 검정 유지

### 1-3. 풋노트 전용 — 초록곰팡이

| 용도 | HEX | RGB | 변수명 |
|---|---|---|---|
| 풋노트 / 출처 | `#006600` | **G102** (R0 G102 B0) | MOLD_GREEN |

- 반드시 **별도 텍스트 박스** 로 슬라이드 하단(y ≈ 18.3cm) 배치
- `#006600` Bold 8pt 고정
- 본 풋노트 외 다른 곳 초록 사용 **금지**

### 1-4. 절대 금지

- 무지개 색·오렌지·빨강·노랑 강조색
- 강조색의 본문 다수 침투
- 그림자·그라데이션·블러 효과

## 2. 폰트 (한·영 자동 분리)

| 분류 | 폰트 |
|---|---|
| 한글 (ea / cs) | **LG스마트체 Regular** |
| 영문·숫자 (latin) | **Arial Narrow** |

한 run 안에 한·영이 섞여도 PowerPoint 가 자동 분리하도록 **반드시 ea / latin 모두 지정** 해야 한다. 본 리포의 기준 구현은 `scripts/build_flexibility_summary.py` 의 `set_run()` 헬퍼.

### 폰트 사이즈 6단 상수 (직접 size=숫자 금지)

```python
SZ_TITLE   = 16   # 슬라이드 타이틀
SZ_BAND    = 12   # 사분면·섹션 라벨 띠
SZ_SECTION = 11   # 카드 내 섹션 헤더 ([적용 방식] 등)
SZ_BODY    = 10   # 본문 bullet, 표 셀, 책임자
SZ_SUB     = 9    # 보조 설명·예시
SZ_FOOT    = 8    # 풋노트 / 출처
```

한 슬라이드에 **4~5개 이내**의 사이즈만 사용. 13pt·14pt 같은 중간값 사용 금지.

## 3. 슬라이드 레이아웃

### 3-1. 슬라이드 사이즈 (LGES 표준)

```python
prs.slide_width  = Cm(27.52)
prs.slide_height = Cm(19.05)
```

### 3-2. 메인 1~2장 + 보조 N장 구조

- **메인**: 임원이 한눈에 의사결정 가능한 요약 슬라이드 (1장 원칙, 최대 2장). 박스 안에 내용이 **반드시 모두** 들어가야 함
- **보조**: 메인에 다 담지 못한 상세를 여러 장으로 분리. 회색조 톤(흰색+회색), 강조 파랑 가능한 미사용. 첫 줄 라벨 띠에 `[보조 N/M] 주제` 표기

### 3-3. 메인 4사분면 표준 레이아웃 (권장)

```
┌───── 타이틀 (16pt B) ────── 우상단: 책임자 (10pt B 파랑) ─────┐
├─────────────────────────────────────────────────────────────┤
│      [핵심 메시지] 박스 (LIGHT_BLUE 배경 + MID_GRAY 보더)    │
├──────────────────────────┬──────────────────────────────────┤
│ ① 라벨 띠 (DIM_GRAY)     │ ② 라벨 띠 (DIM_GRAY)              │
│  카드 내용 (MID_GRAY)    │  카드 내용                       │
├──────────────────────────┼──────────────────────────────────┤
│ ③ 라벨 띠                │ ④ 라벨 띠                        │
│  카드 내용               │  + [결정 포인트] 박스 (파랑 강조)│
├─────────────────────────────────────────────────────────────┤
│ * 출처 : ...  ← 별도 텍스트박스, 초록곰팡이(#006600) 8pt B  │
└─────────────────────────────────────────────────────────────┘
```

**좌표 권장값**

| 영역 | y 시작 | 높이 |
|---|---|---|
| 타이틀 | 0.25 | 1.0 |
| 구분선 | 1.35 | — (MID_GRAY 0.75pt) |
| 핵심 메시지 박스 | 1.55 | 1.95 |
| 상단 사분면 (Q1·Q2) | 3.75 | 7.20 |
| 하단 사분면 (Q3·Q4) | 11.10 | 6.90 |
| 풋노트 | 18.30 | 0.55 |

### 3-4. 표 룰

- 헤더 행: GRAY 배경, 10pt Bold 검정, 가운데 정렬
- 데이터 행: 흰색, 10pt 검정, 본문 좌측·숫자 가운데
- 합계·강조 1행 한정: SOFT_GRAY 배경, **수치만 파랑 Bold**
- 메인 표 최대 6~7행. 더 길면 보조 슬라이드로 분리

## 4. 헬퍼 함수 (재사용 권장)

본 리포의 `scripts/build_flexibility_summary.py` 또는 `scripts/build_skills_review_summary.py` 의 다음 헬퍼를 신규 PPT 빌더에 그대로 가져와 쓸 것:

| 헬퍼 | 역할 |
|---|---|
| `set_run(run, text, ...)` | run 1개에 한·영 분리 폰트·사이즈·색 적용 |
| `write_lines(tf, lines, ...)` | 텍스트프레임 전체를 paragraph 리스트로 채움 |
| `add_rect(slide, l, t, w, h, fill=, line=)` | 직사각형 도형 (그림자 제거 포함) |
| `add_text(slide, l, t, w, h, lines, ...)` | 텍스트 박스 + 본문 |
| `add_label_band(slide, l, t, w, text)` | 사분면 라벨 띠 (DIM_GRAY + 흰글) |

## 5. PR 머지 전 체크리스트

**구조**
- [ ] 메인 슬라이드는 **1~2장** 이내, 임원 한눈 요약
- [ ] 메인의 모든 텍스트·표가 박스 안에 완전히 들어감
- [ ] 메인에 다 못 담은 내용은 보조 슬라이드 N장으로 분리
- [ ] 보조 슬라이드 첫 줄에 `[보조 N/M]` 표기

**디자인**
- [ ] 슬라이드 사이즈 27.52 × 19.05 cm
- [ ] 폰트는 LG스마트체(한글) + Arial Narrow(영문), run-level 분리
- [ ] 사이즈 상수 `SZ_TITLE/SZ_BAND/SZ_SECTION/SZ_BODY/SZ_SUB/SZ_FOOT` 만 사용
- [ ] 파랑 강조는 메인에서 **3~5개 포인트 이내**, 보조는 회색·검정
- [ ] 풋노트 = **별도 텍스트박스 + `#006600` Bold 8pt**
- [ ] 일반 섹션 헤더/중요 포인트는 검정 Bold (색 변경 없음)
- [ ] 텍스트 색은 검정 또는 회색 계열만 (그 외 색은 파랑·풋노트 초록만)
- [ ] 모든 박스·카드 외곽선은 검정 또는 회색만
- [ ] 박스 안 내용이 카드 영역을 벗어나지 않음

## 6. DOCX / MD 산출물 룰

본 스킬은 PPT 가 중심이지만 **DOCX/MD** 산출물에도 다음 룰이 적용된다.

### DOCX

- 본문 한글: 맑은 고딕 또는 LG스마트체 / 영문: Arial Narrow
- 헤더 1: 검정 Bold 14pt / 헤더 2: 검정 Bold 12pt / 본문: 10~11pt
- 출처 인용은 본문 하단 푸터에 `(출처: ...)` 형식

### MD

- 모든 외부 인용은 `(출처: <도메인/파일명>)` 또는 `참조: references/<카테고리>/<파일>` 또는 `자료 미확인` 형식 — `scripts/hooks/check_output_citations.py` 룰과 일치
- 표는 GitHub-flavored markdown
- 강조는 **Bold** 만 사용 (색 변경 불가, MD 한계)
