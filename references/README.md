# references/ — 참조 자료 (Reference Materials)

에이전트가 작업 시 참조하는 자료(로드맵, 정책, 과거 회의록, 기술자료, 용어집)를 담는 폴더입니다.

`templates/`(보고서 양식) 와 `sample_data/`(수치 데이터) 와 구분됩니다.

| 폴더 | `templates/` | `sample_data/` | **`references/`** |
|---|---|---|---|
| 용도 | 빈칸 채울 양식 | 수치 데이터 (KPI 등) | **컨텍스트·근거 자료** |
| 사용 에이전트 | document-writer, ppt-writer | data-teammate, ops-teammate | **모든 에이전트** |
| 파일 형식 | .md (placeholder), .pptx | .md, .csv, .json | .md (어떤 형식이든 OK) |

---

## 자료 추가는 정말 간단합니다

> **.md 파일을 적당한 폴더에 던져 넣기만 하면 됩니다.**
> 제목·카테고리·날짜·요약은 스크립트가 자동으로 인식합니다.

1. **카테고리에 맞는 하위 폴더에 두기** (어디 둘지 헷갈리면 그냥 `references/` 직속에 두세요. 자동으로 "기타" 분류됩니다)
2. 끝. 다음 세션이 시작되면 자동으로 인덱스에 등록됩니다.

### 카테고리 선택 가이드

| 자료 성격 | 어느 폴더로? |
|---|---|
| "앞으로 우리가 뭘 할 거다" | `roadmap/` |
| "이런 기준·규정이다" | `policy/` |
| "지난 ○○○ 회의에서 결정됨" | `과거회의록/` |
| "외부에서 본 자료·논문 요약" | `기술자료/` |
| "CATL·SDI 등 경쟁사 동향" | `경쟁사/` |
| "이 약어가 무슨 뜻이냐" | `용어집/` |
| 어디 둘지 모르겠음 | `references/` 직속 → "기타" 자동 분류 |

### 카테고리별 권장 자료 (앞으로 추가하면 좋은 것들)

| 카테고리 | 권장 자료 |
|---|---|
| `roadmap/` | 연도별 사업계획, 분기별 마일스톤 업데이트, 신규 기술과제 로드맵 |
| `policy/` | 투자 승인 기준, 협력사 등록·평가 기준, 안전·환경 정책, 데이터 보안 정책 |
| `과거회의록/` | 분기별 R&D 리뷰, 그룹장 정례 미팅 결정사항, 양산 라인 검토 회의 |
| `기술자료/` | NVIDIA Isaac SIM/Omniverse 사용 가이드, AMR 벤더 자료, 논문·백서 요약, MOMA/SMC 등 핵심 기술 상세 |
| `경쟁사/` | CATL/SDI/BYD 분기별 동향, 글로벌 셋업 자동화 트렌드, 산업 표준 동향 |
| `용어집/` | 사내 약어 사전, 산업 전문용어, 솔루션 명칭 |

권장이지 필수가 아닙니다. 가지고 있는 자료부터 차근차근 채우면 됩니다.

## 더 정확하게 인식되게 하려면 (선택 사항)

자동 인식이 100% 정확하지 않을 수 있어요. 더 정확한 결과를 원하면:

| 원하는 것 | 이렇게 두세요 |
|---|---|
| **제목** 을 정확히 | 파일 첫 줄을 `# 정식 제목` 으로 시작 |
| **날짜** 를 정확히 | 파일명을 `2026-05-22_제목.md` 식으로 (또는 `20260522`도 OK) |
| **요약** 을 정확히 | 제목(`#`) 다음 줄에 한두 문장으로 요약 작성 |

대부분의 경우 위 3가지만 신경 쓰면 충분합니다.

## 인덱스 갱신 (자동/수동 둘 다 지원)

스크립트가 모든 자료를 스캔해서 두 파일을 만듭니다:

- `references/INDEX.md` — 사람이 보는 목차 (자동 생성, 수동 편집 금지)
- `references/manifest.json` — 에이전트가 보는 인덱스 (자동 생성)

### 갱신 방법

| 언제 | 어떻게 |
|---|---|
| Claude Code 세션을 새로 열 때 | 자동으로 갱신됩니다 (SessionStart hook) |
| 자료를 추가하고 같은 세션에서 즉시 반영하고 싶을 때 | `python3 scripts/build_reference_index.py` |
| GitHub 웹 UI에서 main에 자료 업로드한 직후 | Claude에게 "방금 올린 자료 인덱스에 반영해줘" 라고 한 마디 → 새 PR로 정리됩니다 |

## 자료 인용 형식 (에이전트가 사용)

에이전트가 references 자료를 인용할 때 다음 형식을 씁니다.

```
references/roadmap/2026_FA기술담당_중장기로드맵_v2.md [12행]
```

자료가 없으면 "관련 references 자료 없음"이라고 명시하고 추측하지 않습니다.

## 고급 옵션 (선택 사항)

frontmatter라는 메타데이터 블록을 파일 상단에 넣으면 자동 추출보다 우선합니다. **반드시 필요한 건 아니에요.**

```yaml
---
title: FA기술담당 중장기 기술 로드맵 v2
category: roadmap
date: 2026-05-22
owner: FA기술담당
tags: [모듈러, 셋업자동화, AX, PhysicalAI]
summary: 그룹장 코멘트 반영, 유연성·재활용성 지표 신설
related:
  - references/roadmap/2026_FA기술담당_중장기로드맵.md
---
```

- `tags`: 자동 추출되지 않습니다. 검색·필터링이 중요한 자료에만 직접 넣으세요.
- `related`: 관련 자료가 있을 때만 넣으세요.
- 그 외 필드는 모두 자동 추출됩니다.
