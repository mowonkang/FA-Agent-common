# 분류형 보고 레이아웃 표준 (사내 보고양식 기반)

## 1. 배경 및 적용 범위

- **배경**: 기존 2×2 사분면(quadrant) 레이아웃(`quadrant()` 함수 계열)은 시선이 4분면으로
  분산되어 "눈에 안 들어온다"는 피드백이 많았다. 사내 보고양식
  `templates/사내양식/(보고양식)YYMMDD_보고제목_v1.0.pptx` **슬라이드 1** 의 좌측 분류 레일 +
  우측 내용 구조로 전환해 위→아래 읽기 흐름을 확보한다.
- **적용 범위**: FA기술담당의 모든 보고용 `.pptx` 산출물(정의·현황·경쟁사 분석·검토·임원 보고 등).
  녹취록 회의록(meeting-minutes 스킬) 등 별도 양식이 정해진 산출물은 제외.
- **상위 규칙**: 본 표준은 `templates/사내양식/LGES_PPT_작업_가이드.md` (색·폰트·풋노트 룰) 위에서
  동작한다. 충돌 시 가이드의 색·폰트·사이즈 룰이 우선한다.

## 2. 레이아웃 정의 (슬라이드 1 기준)

| 요소 | 규격 |
|---|---|
| 슬라이드 | 27.52 × 19.05 cm (10.83 × 7.5 in) |
| 제목 | 좌측 상단, 검정 Bold `SZ_TITLE`(16pt) |
| 조직/날짜(byline) | 우측 상단, **검정** Bold, `조직명` + `(YYYY.MM.DD)` 2줄 |
| Head message | 제목 아래, **배경 없는 검정** Bold `SZ_HEAD`(15pt) **서술형 1~2줄(1줄 선호)** |
| 좌측 분류 레일 | 분류 1~N, 무채색 테두리(검정), 라벨 Bold·세로 중앙·폰트 자동 맞춤·자유 줄바꿈, **높이는 우측 내용 분량에 맞춰 자동 가변** |
| 우측 내용 | 분류별 세션 메시지(`SZ_SECTION` Bold) + 상세 내용(`SZ_BODY`) + 선택 표/이미지 박스 |
| 풋노트 | 별도 텍스트박스, 초록 `#006600` Bold `SZ_FOOT`(8pt), 출처 명시 |

- 분류 개수는 **1~6 자유**. 3단(개요·내용·결론) 구성도 허용.
- 메인은 모든 내용이 슬라이드 안에 수렴해야 하며(무오버플로 필수), 넘치는 상세는 보조로 분리한다.

## 3. 구현 — 재사용 헬퍼 `scripts/_ppt_layout.py`

메인 슬라이드는 `render_classified_slide()` 로 그린다. 드로잉 프리미티브
(`set_run`/`write_lines`/`add_rect`/`add_text`/`add_label_band`/`fill_table`/`title_block`/
`add_footnote`)와 색·폰트·`SZ_*` 상수도 본 모듈에서 import 한다(빌더별 중복 정의 금지).

```python
render_classified_slide(
    slide, *, title, head_message, sections,
    footnote=None, byline=None,
    content_top=3.10, content_bottom=18.20, fill_to_bottom=True)
```

- `head_message` : `write_lines` 형식 리스트. 검정 Bold 15pt 서술형.
- `byline` : 문자열 또는 리스트(`["FA기술담당", "(2026.05.27)"]`).
- `sections[i]` dict:
  `label`(좌측 박스), `session_message`(우측 헤더), `content`(write_lines 리스트),
  `table`(`{headers, rows, col_w, row_h, body_size?, head_size?}`)|None,
  `image_box`(cm 높이)|None, `height`(명시 override cm)|None, `label_size`(override)|None.
- **자동 높이 맞춤**: 각 분류 박스 높이는 우측 내용(줄 수·표·이미지)에서 결정론적으로 산정.
  총합이 영역을 넘으면 스케일 다운(`scale<0.80` 시 stdout WARNING), 남으면 `fill_to_bottom`
  으로 하단까지 비례 분배.
- **표 폰트**: 기본 `SZ_BODY`(10pt). 7행 이상 밀집 매트릭스 등 불가피한 경우만 `SZ_SUB`(9pt).

보조 슬라이드는 **표 중심** 으로 정보밀도를 높여도 되며(동일 헬퍼의 `title_block`/
`add_label_band`/`fill_table`/`add_text`/`add_footnote` 사용), 전체 톤(색·폰트·풋노트)은 통일한다.

## 4. 작성 워크플로우 (SOP)

1. **데이터 수집 — 추측 금지·출처 명시**: 수치·일정은 `references/`·`outputs/`·`sample_data/`
   에서 확보하고 풋노트에 `파일명 [N행]` 으로 인용. 자료 없으면 "자료 미확인"으로 명시.
2. **메인 = 분류형 요약**: 1장에 핵심 결론(서술형 Head) + 분류 1~N 요약. 모든 내용 박스 내 수렴.
3. **보조 = 표 중심 심층**: 상세 매트릭스·시나리오·근거를 보조 N장으로 분리. 메인에서 "→ 보조 N 참조".
4. **프로 엔지니어 톤**: 내용은 항상 기술적·전문가 수준으로 서술(정의·산식·공정·정량 근거).
5. **방향성 확인**: 보조 장수·구성, 강조 포인트, 설명 방식 등 **판단이 갈리는 지점은 작성 전
   `AskUserQuestion` 으로 방향을 확정** 받는다(임의 확대 금지).
6. **검증**: 빌더 실행 → 오버플로 WARNING 확인 → 재오픈 자기검증(박스 높이·정렬·표 위치).
   시각 최종 확인은 PowerPoint 권장(렌더 환경에 따라 LibreOffice 변환이 막힐 수 있음).

## 5. 기준 구현 / 예시

- 헬퍼: `scripts/_ppt_layout.py`

  → `outputs/분류형_레이아웃_데모.pptx`
- 정의 자료(메인 1장): `scripts/build_flexibility_definition.py`
  → `outputs/유연성지표_정의_2026-05-27.pptx`
- 경쟁사 분석(메인 + 보조 3장): `scripts/build_flexibility_competitor_compare.py`
  → `outputs/유연성_SDI_CATL_비교분석_2026-05-27.pptx`

- 텍스트 무오버플로 · 임원 즉시 활용 품질 · 스토리는 필수.
